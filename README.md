# 智能二手商品发布助手 - 项目总体设计说明书

## 1. 项目背景与业务目标

在主流二手交易平台（如闲鱼、转转）中，商品发布的门槛较高。用户往往面临不知如何描述商品成色、缺乏撰写吸引力文案的技巧、以及对二手市场行情定价不准等痛点。

本项目旨在构建一个 C 端智能辅助工具。通过接入多模态大语言模型（Vision-LLM）与传统数据库行情系统的深度融合，将原本繁琐的"手动填表+查价+构思文案"的流程，极简至 **"上传图片 → AI 识别 → 用户确认 → 自动生成"** 的操作闭环，极大降低普通用户的发布门槛。

## 2. 系统核心架构

系统采用前后端分离的单体应用架构，以 FastAPI 原生异步（`async/await`）能力驱动 AI 推理长耗时任务与 SSE 流式输出的流畅体验。

- **前端交互层 (Client Tier)：** 负责用户交互界面展现、登录/注册表单、图片上传预览与浏览器端压缩、用户确认/修正 AI 识别结果的中间页、历史记录管理、商城浏览、以及接收流式（SSE）响应实现打字机渲染效果。
- **网关与调度层 (API Gateway / Backend)：** 核心胶水层，负责接收前端请求、文件本地落盘、用户认证鉴权、统一编排大模型工作流（视觉识别→查价→文案生成）、解析结构化数据以及与数据库 CRUD 交互。
- **模型服务层 (AI Engine)：**
  - 视觉感知模块（Qwen-VL-Max）：将图片转化为结构化文本（品类、品牌、型号、成色）。
  - 文本生成模块（DeepSeek-V4-Pro）：综合视觉识别结果与行情数据，流式生成带货文案与定价策略。
  - 统一通过策略模式抽象 LLM 客户端，便于扩展与切换模型。
- **数据持久层 (Data Tier)：** 存储系统元数据（用户 + PBKDF2 密码哈希）、二手行情基准表、商品发布记录以及全链路 AI 调用审计日志。图片资产基于本地文件系统存储。

## 3. 核心功能模块与数据流转

**整体流程**

```
用户注册/登录 ──► 上传图片 ──► AI 视觉识别 ──► 数据库查价 ──► 用户确认页（可修正）
                                                              ├── 存草稿
                                                              │ 确认
                                                     AI 流式生成文案（SSE 打字机）
                                                              │
                                                     落库 published_items
                                                     + ai_audit_logs 审计
                                                              │
                                                     商城展示（全用户）
```

### 3.1 用户认证模块

- **行为：** 用户注册（用户名 + 密码 + 确认密码）或登录。
- **安全：** PBKDF2-SHA256（100,000 迭代 + 16 字节随机 salt）哈希存储密码，数据库中不存明文。
- **路由守卫：** Vue Router 全局前置守卫，未登录用户访问发布/历史页自动跳转登录页。

### 3.2 图像预处理与感知模块

- **行为：** 用户上传 1-3 张二手物品照片。前端在浏览器端进行 Canvas 压缩后提交。
- **流转：** 图片发送至后端，后端先将其保存在 `static/uploads/` 本地目录，随后转为 Base64 Data URI 调用 Qwen-VL-Max，通过 Few-shot + Schema 强约束 Prompt 迫使模型输出结构化 JSON。
- **输出：** 返回标准 JSON 对象：`{"category": "手机", "brand": "Apple", "model": "iPhone 13", "condition": "9成新，屏幕有细微划痕"}`。
- **容错：** Python 端使用 `json.loads` + Pydantic 校验做双重保障；解析失败则向前端返回友好错误提示。

### 3.3 关系型数据联动模块（查价引擎）

- **行为：** 接收上游模块解析出的品牌（brand）和型号（model）。
- **流转：** 后端采用**三层匹配策略**查询 MySQL `market_prices` 表：
  1. 精确匹配（`WHERE brand=X AND model=Y`）
  2. 未命中 → Python `difflib.SequenceMatcher` 计算相似度，取最高且 > 0.7 的结果
  3. 仍未命中 → 返回空，文案中标注"暂无市场行情数据"
- **输出：** 历史均价、最高价、最低价区间，作为文案生成的上下文参考。

### 3.4 用户确认中间页 + 输入校验

- **行为：** AI 识别和查价结果返回后，**并不直接生成文案**，而是先展示一个确认页。
- **功能：** 用户可查看并**手动修改**识别出的品类、品牌、型号、成色描述。修改后的数据重新触发查价。
- **三级校验体系：** ok（绿框通过）/ warn（黄框提示但允许继续）/ error（红框拦截，阻止发布）。
- **草稿功能：** 信息不完整可点击「💾 存草稿」，后续从历史页发布。
- **意义：** 大幅提升可用性，避免 LLM 识别错误导致最终文案不可用。也是答辩中的"人机协作"亮点。

### 3.5 智能文案与定价生成模块

- **行为：** 用户在确认页点击"生成文案"后，将多维数据融合并请求文本生成。
- **流转：** 后端组装 System Prompt（设定为二手带货专家）及 User Prompt（填入用户确认后的物品 JSON 特征与行情数据），请求 DeepSeek-V4-Pro。
- **输出：** 采用 Server-Sent Events (SSE) 协议，将商品标题、分段式正文及建议售价，以流式传输返回给前端渲染（打字机效果）。

### 3.6 业务留存与审计模块

- **行为：** 生成资产的持久化与系统状态记录。
- **流转 & 输出：** 每次完整的生成流程结束后，后端触发异步事务：
  1. 向 `published_items` 表存入最终的商品文案、定价及图片本地路径。
  2. 向 `ai_audit_logs` 表插入 JSON 原生报文、执行耗时、报错信息的 API 执行记录。

### 3.7 二手商城模块

- **行为：** 所有用户已发布商品在商城统一展示。
- **功能：** 分类下拉筛选 + 关键词实时搜索，无需登录即可浏览。
- **数据：** JOIN `users` 获取卖家名，仅展示 `status='published'` 的商品。

## 4. 技术栈选型

| **层级**       | **技术/框架选型**                 | **选型依据**                                                     |
| -------------- | --------------------------------- | ---------------------------------------------------------------- |
| **前端**       | Vue 3 + Vite + TailwindCSS        | 轻量级响应式界面，HMR 开发体验佳，组件化效率高。                 |
| **后端**       | FastAPI (Python 3.11+)            | 原生异步（`async/await`），完美适配 AI 长耗时推理与 SSE 流式输出。 |
| **数据库**     | MySQL 8.0                         | 成熟可靠，支撑前期几十万级商品基准表与日志表。                   |
| **认证安全**   | PBKDF2-SHA256 (100,000 迭代)      | 内置 hashlib 实现，零外部依赖，校园项目答辩加分项。              |
| **视觉模型**   | Qwen-VL-Max（阿里云百炼）          | 中文场景视觉理解最优，结构化 JSON 输出稳定，Base64 Data URI 传图。 |
| **文本模型**   | DeepSeek-V4-Pro（DeepSeek 官方）   | 中文带货文案质量顶级，流式输出速度快。                           |
| **AI 客户端**  | 策略模式抽象 `BaseLLMClient`       | 统一封装，便于切换/扩展不同模型服务商。                          |
| **图片存储**   | 本地文件系统 + FastAPI StaticFiles | 开发阶段最简方案，直接挂载 `static/uploads/` 目录。              |

## 5. API 接口一览

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/v1/register` | 用户注册 | 无需 |
| POST | `/api/v1/login` | 用户登录 | 无需 |
| POST | `/api/v1/extract` | 上传图片 + AI 视觉识别 | 需登录 |
| GET  | `/api/v1/price` | 查询市场行情 | 需登录 |
| POST | `/api/v1/generate` | SSE 流式生成文案 | 需登录 |
| GET  | `/api/v1/history` | 查询发布历史 | 需登录 |
| POST | `/api/v1/history/save` | 保存发布记录 | 需登录 |
| POST | `/api/v1/history/{id}/delist` | 下架商品 | 需登录 |
| POST | `/api/v1/history/{id}/publish` | 发布草稿 | 需登录 |
| GET  | `/api/v1/market` | 商城列表（支持 keyword + category 搜索） | 无需 |

## 7. 项目落地实施路径 (Milestones)

1. **Phase 1：数据基建 (Database & Mock Data)。** 设计 MySQL 核心业务表（`users`, `market_prices`, `published_items`, `ai_audit_logs`），手动填入 20+ 条校园常见二手物品的基准价格数据，初始化 PBKDF2 密码哈希。
2. **Phase 2：后端 API 骨架与模型接入。** 使用 FastAPI 搭建接口，封装 LLM 客户端抽象层，实现用户认证系统，分别跑通 Qwen-VL-Max 的结构化输出和 DeepSeek-V4-Pro 的流式生成。
3. **Phase 3：业务流编排。** 在后端实现"图片识别 → 查价 → 用户确认 → 生成文案 → 日志落库"的完整连贯逻辑。
4. **Phase 4：前端交互联调。** 编写 Vue 3 前端页面，实现登录/注册、图片上传预览、确认修改中间页（含三级输入校验）、SSE 流式文案打字机效果、发布历史管理（草稿/发布/下架）以及二手商城浏览。
