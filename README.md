1. # 智能二手商品发布助手 - 项目总体设计说明书

   ## 1. 项目背景与业务目标

   在主流二手交易平台（如闲鱼、转转）中，商品发布的门槛较高。用户往往面临不知如何描述商品成色、缺乏撰写吸引力文案的技巧、以及对二手市场行情定价不准等痛点。

   本项目旨在构建一个 C 端智能辅助工具。通过接入多模态大语言模型（Vision-LLM）与传统数据库行情系统的深度融合，将原本繁琐的“手动填表+查价+构思文案”的流程，极简至**“上传图片 -> 自动生成”**的单步操作，极大降低普通用户的发布门槛。

   ## 2. 系统核心架构

   系统采用前后端分离、轻量级异步微服务架构，以确保 AI 推理任务的稳定运行与前端交互的流畅度。

   - **前端交互层 (Client Tier)：** 负责用户交互界面展现、图片上传预处理（本地极速压缩）、以及接收流式（SSE）响应实现打字机效果。
   - **网关与调度层 (API Gateway / Backend)：** 核心胶水层，负责接收前端请求、执行文件本地落盘、编排大模型工作流、解析结构化数据以及与数据库进行 CRUD 交互。
   - **模型服务层 (AI Engine)：** - 视觉感知模块：负责将图片转化为结构化文本（品牌、型号、成色）。
     - 文本生成模块：负责综合参数生成最终的营销文案与定价策略。
   - **数据持久层 (Data Tier)：** 存储系统元数据、二手行情参考基准表、全链路安全审计日志，**以及基于本地相对路径存储的商品图文发布资产**。

   ## 3. 核心功能模块与数据流转

   ### 3.1 图像预处理与感知模块

   - **行为：** 用户上传 1-3 张二手物品照片。
   - **流转：** 图片发送至后端，后端先将其保存在本地静态目录。随后调用多模态模型（Vision-LLM），并强制通过 Schema 限定输出格式。
   - **输出：** 返回标准的 JSON 对象，包含：品类（Category）、品牌（Brand）、型号（Model）、成色瑕疵（Condition），及图片的本地路径。

   ### 3.2 关系型数据联动模块（查价引擎）

   - **行为：** 接收上游模块解析出的品牌和型号数据。
   - **流转：** 后端动态拼接 SQL 语句，查询内部 MySQL 数据库的 **`market_prices`** 表。
   - **输出：** 提取该商品的历史均价、最高价、最低价区间，作为下一步的上下文参考。

   ### 3.3 智能文案与定价生成模块

   - **行为：** 将多维数据融合并请求文本生成。
   - **流转：** 后端组装 System Prompt（设定为二手带货专家）及 User Prompt（填入上游传入的物品 JSON 特征与行情数据），请求文本大模型。
   - **输出：** 采用 Server-Sent Events (SSE) 协议，将生成的极具吸引力的商品标题、分段式正文及建议售价，以流式传输返回给前端渲染。

   ### 3.4 业务留存与审计模块

   - **行为：** 生成资产的持久化与系统状态记录。
   - **流转 & 输出：** 每次完整的生成流程结束后，后端触发异步事务：
     1. 首先向 **`published_items`** 表存入最终的商品带货文案、定价及**图片的本地物理路径**。
     2. 随后向 **`ai_audit_logs`** 表插入包含 JSON 原生报文、执行耗时、报错信息的 API 执行记录。

   ## 4. 技术栈选型建议

   | **层级**       | **技术/框架选型**                   | **选型依据**                                                 |
   | -------------- | ----------------------------------- | ------------------------------------------------------------ |
   | **前端**       | Vue 3 + TailwindCSS                 | 适合快速搭建响应式界面，组件化开发效率高。                   |
   | **后端**       | FastAPI (Python)                    | 原生支持异步（`async/await`），完美契合 AI 推理长耗时接口与 SSE 流式输出。 |
   | **数据库**     | MySQL 8.0                           | 成熟可靠的关系型数据库，足以支撑前期几十万级的商品基准表与日志表。 |
   | **大模型 API** | Qwen-VL (视觉) + DeepSeek-V3 (文本) | 国内生态支持良好，响应速度快，API 调用成本极低，适合个人开发者与独立项目落地。 |

   ## 5. 项目落地实施路径 (Milestones)

   1. **Phase 1：数据基建 (Database & Mock Data)。** 设计 MySQL 核心业务表（`users`, `market_prices`, **`published_items`**, **`ai_audit_logs`**），并手动填入部分测试商品的基准价格数据。
   2. **Phase 2：后端 API 骨架与模型接入。** 使用 FastAPI 搭建接口，分别跑通 Vision-LLM 的结构化输出和 Text-LLM 的流式输出。
   3. **Phase 3：业务流编排。** 在后端实现从“图片识别 -> 查询数据库 -> 生成文案 -> 日志落库”的连贯逻辑。
   4. **Phase 4：前端交互联调。** 编写前端页面，对接 API，打磨图片上传动画与文案流式生成的“打字机”体验。
