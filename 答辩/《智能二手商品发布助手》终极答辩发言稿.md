# 《智能二手商品发布助手》终极答辩发言稿

## 开场致辞

尊敬的各位评委老师，大家好。我是第一组的答辩人XXX。今天我们团队为您汇报的毕业设计项目是：《智能二手交易发布系统——基于多模态 AI 与异步架构的无感发布解决方案》。  

今天的答辩，我将按照大屏幕上的七步逻辑闭环，从背景痛点出发，一路为您解剖我们系统的核心功能、底层架构，以及我们在工程落地中真实的踩坑与复盘。  

## 第一部分：痛点与破局

在项目初期，我们调研发现，传统的二手交易平台（如闲鱼、转转）仅仅解决了商品“展示”的问题，却把繁琐的“发布”全丢给了用户。这导致了阻碍交易的四大核心痛点：  

第一，**发布流程极度繁琐**，用户需要耗费大约 20 分钟反复拍照、对比参数并填写表单。 第二，**定价盲区**，个人卖家对行情判断不准，标高了卖不掉，标低了自己吃亏。 第三，**描述词穷**，普通用户根本不知道怎么提炼数码产品的核心参数与卖点。 第四，**瑕疵漏判**，肉眼极易遗漏屏幕划痕或机身磕碰，导致后期严重的信任裂缝和交易纠纷。  

基于这四大痛点，我们确立了本项目的核心使命：**把 20 分钟的繁琐操作，压进 1 分钟的无感体验里，彻底消除二手交易的摩擦力**。  

## 第二部分：核心功能与生态构建

为了实现这个使命，请各位老师看大屏幕我们的核心产品交互。

在我们的系统中，用户只需要做一个极简的动作：**拖拽上传一张实拍图，点击识别**。接下来，我们的系统会在 1 分钟内完成全自动接管。  

系统不仅会自动识别商品的大类、品牌与型号，还会立刻调用 YOLO 模型，在实拍图上用红框精准标注出瑕疵破损区域。紧接着，结合历史价格走势，系统会给出合理的建议售价和扣款明细。  

最核心的体验在于最后一步的**流式文案生成**。大家可以看到大屏幕上的这段文字，我们结合了 DeepSeek 大模型和 SSE 长连接技术，让这 200 字的专业带货文案以“打字机”的形式逐字呈现在用户眼前。这彻底消除了用户在等待大模型响应时的白屏焦虑，用户觉得满意，一键复制即可发布。  

此外，我们不仅做了一个单点工具，还构建了完整的商业生态，包括买家侧的“以图搜图”、以及后台的系统全局大屏统计、违规商品审核与每周模型训练控制台。  

## 第三部分：全局架构与数据梳理

为了支撑上述的丝滑体验，我们严格遵循现代工程规范，设计了一套 **4 层异构协同架构**。  

在最外围的**接入层**，我们使用了 Cloudflare Tunnel 实现了内网穿透与服务器 IP 隐藏，实现了零公网 IP 的安全部署。 在核心的**应用层**，系统大脑采用了 FastAPI 异步框架，并且我们在全链路的数据库查询和文件操作上，采用了 `aiomysql` 和 `aiofiles` 进行彻底的异步化改造，以支撑高并发吞吐。 在**AI 调度层**，我们采用了“云边端协同”策略：在本地物理机环境原生运行 YOLOv8、CLIP 特征提取和 rembg 去噪模型；同时通过 API 远程并发调度云端的 Qwen 和 DeepSeek 大脑。 底层的**数据层**，除了 MySQL 处理事务外，我们还通过 Docker 容器化部署了 Qdrant 向量数据库，用于 512 维特征的高速检索。  

大家请看右侧的业务流程时序图，当这套架构运转时：图片经过预处理后，会立刻开启“三线异步并发推理”。这其中最精妙的是我们的**三层智能查价策略**：系统会优先在 24 小时缓存中检查；未命中则触发 OneBound API 爬虫抓取实时电商数据；如果爬取失败，系统还会自动回退到数据库进行相似度模糊匹配兜底，确保高可用性。  

## 第四部分：算法与应用

支撑这套管线的，是我们打磨的三大算法基石。

首先是基于 **YOLOv8 的端侧极速推理**。它能在本地瞬间对 7 大类 Kaputt 工业标准的缺陷（如变形、溢漏、表面瑕疵）进行定位，为后续的语义裁决提供绝对客观的“视觉先验事实”。 其次是结合 **CLIP 与 Qdrant 的多模态检索**。实现了跨越文本的毫秒级近似最近邻（HNSW）图像相似度比对。 最后，是我们独创的**四因子智能定价公式**。最终定价 = 市场均价 × 瑕疵折损系数 × 贬值因子 × 波动因子。我们甚至加入了根据苹果或安卓品牌计算的“月度贬值率”，以及正弦波模拟的“周期波动噪声”，让 AI 生成的价格曲线极具真实感。  

## 第五部分：实战复盘与工程踩坑

纸上谈兵终觉浅。在落地这上万行代码的过程中，我们遭遇了六大工程陷阱，在这里向各位老师复盘我们的破局策略：

1. **环境依赖雪崩**：起初团队设备各异，导致 CUDA 冲突严重。我最终编写了 `torchzidong.py` 硬件自适应脚本，实现了零人工干预的开箱即用。  
2. **跨语种逻辑死锁**：YOLO 输出英文，大模型输出中文，传统 if-else 完全失效。我们果断引入大模型作为“零样本裁判”，通过语义层进行对齐纠偏。  
3. **置信度陷阱**：我们发现 YOLO 默认 0.5 的阈值拦截了大量有效瑕疵。最终我们将阈值压低到 0.2 保障高召回率，将降噪任务后置交给了云端大脑。  
4. **算力博弈**：全本地算力不够，全云端 API 太贵。逼出了我们现在的“云边端协同”管线。  
5. **反爬风控**：自研爬虫屡遭 IP 封禁，我们选择了工程妥协，全面拥抱合规的 OneBound 淘宝开放 API，换取了绝对稳定的 SLA。  
6. **异步阻塞**：传统的同步请求拖垮了系统并发。我们对全链路进行了 `aiohttp` 与 `aiomysql` 的彻底异步重构，释放了服务器潜能。

## 第七部分：亮点、缺陷与未来展望

站在答辩的讲台上，我想总结本项目的两大工程亮点： 一是我们采用了**面向接口的策略模式**，实现了异构大模型的零侵入无缝热插拔。 二是我们构建了**数据飞轮**，低置信度的错题会自动回流，让本地模型摆脱大厂依赖，实现越用越聪明的自我生长。  

当然，系统目前在算力彻底离线和瑕疵粒度上仍有局限。关于未来，我们制定了三步走战略：第一步，推进本地算力的彻底私有化隔离；第二步，打通跨电商平台的全网同步发布中枢；第三步，利用 React Native 开发移动原生端，实现随时随地即拍即发。  

从 20 分钟的繁琐操作，到 1 分钟的无感发布，我们团队用上万行代码完成了一场对交易摩擦力的破壁实验。
