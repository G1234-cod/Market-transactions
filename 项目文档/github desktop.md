一、核心概念
1.1 什么是分支？
分支就是代码的不同版本线。你们两个人各自在自己的分支上开发，互不干扰。

1.2 三个重要分支
分支	谁在用	用途	能直接改吗
main	两个人共用	存放稳定可用的代码	❌ 不能，只能通过合并
feature/NXF-LGTM	你	你的开发分支	✅ 随便改
feature/G1234-cod	队友	队友的开发分支	✅ 随便改
1.3 分支关系图
text
main（稳定主干，你们共用）
  │
  ├── feature/NXF-LGTM（你的开发分支）
  │
  └── feature/G1234-cod（队友的开发分支）
二、分支策略
2.1 核心原则
永远不要直接在 main 分支上修改代码

永远在自己的开发分支上提交和推送

每天开始工作前，先从 main 同步最新代码

2.2 分支命名规则
你的分支：feature/NXF-LGTM

队友的分支：feature/G1234-cod

格式：feature/你的名字

三、基本操作
3.1 打开项目
方式一：从 GitHub Desktop 打开

打开 GitHub Desktop

点击左上角 当前仓库，选择 Market-transactions

点击 存储库 → 在外部编辑器中打开（或按 Ctrl + Shift + A）

方式二：从 Trae 直接打开文件夹

打开 Trae

点击 打开文件夹

选择项目目录（如 D:\GitHub\Market-transactions-main）

3.2 查看当前在哪个分支
在 GitHub Desktop 顶部，当前分支 旁边显示的就是你所在的分支名称。

显示	含义
main	你在主分支
feature/NXF-LGTM	你在自己的开发分支
feature/G1234-cod	你在队友的开发分支
3.3 切换分支
点击顶部 当前分支 下拉菜单 → 选择你要切换到的分支。

注意： 切换后，本地文件会自动切换成该分支的版本。

3.4 创建新分支
点击顶部 当前分支 → 新建分支

名称 输入分支名（如 feature/你的名字）

来源 选择 main

点击 创建分支

点击 发布分支（推送到 GitHub）

3.5 提交更改（每天最常用的操作）
在 Trae 中修改代码，保存文件（Ctrl + S）

切换回 GitHub Desktop

左侧 更改 列表会显示所有修改过的文件

勾选你要提交的文件（或全选）

在 摘要 框中写清楚本次修改内容

点击 提交到 [你的分支名]

点击顶部 推送 origin

3.6 拉取最新代码
操作： 点击顶部 拉取 origin（或 Fetch origin → Pull origin）

作用： 把远程仓库（包括 main 和其他分支）的最新改动下载到你的电脑。

3.7 把主分支同步到你分支
当队友合并了代码到 main，你想让你的分支也包含这些更新：

确保当前分支是你自己的（如 feature/NXF-LGTM）

点击 拉取 origin

点击 分支 → 合并到当前分支...

来源选择 main

点击 合并

点击 推送 origin

3.8 把队友分支合并到你分支
当你需要队友的代码来继续开发时：

确保当前分支是你自己的（如 feature/NXF-LGTM）

点击 拉取 origin

点击 分支 → 合并到当前分支...

来源选择队友的分支（如 feature/G1234-cod）

点击 合并

点击 推送 origin

3.9 查看队友的代码（不合并）
点击顶部 当前分支 下拉菜单

选择队友的分支（如 feature/G1234-cod）

浏览文件

查看完后，切回自己的分支继续工作

3.10 合并到主分支
前提： 你的功能完全做好、测试通过后：

点击 当前分支 → 切换到 main

点击 拉取 origin

点击 分支 → 合并到当前分支...

来源选择你的开发分支（如 feature/NXF-LGTM）

点击 合并

点击 推送 origin

3.11 解决冲突
什么时候会发生？ 你们俩改了同一个文件的同一行代码。

冲突文件长什么样：

text
<<<<<<< HEAD
你的代码
=======
队友的代码
>>>>>>> feature/G1234-cod
解决步骤：

在 Trae 中打开冲突文件

决定保留哪个版本（或自己重写）

删掉所有 <<<<<<<、=======、>>>>>>> 标记

保存文件

回到 GitHub Desktop，勾选冲突文件

点击 提交合并

点击 推送 origin

四、日常节奏
4.1 每天开始工作前
text
拉取 origin → 把 main 合并到你分支 → 开始写代码
4.2 白天写代码
text
写代码 → 保存 → 提交 → 推送（反复循环）
4.3 需要队友代码时
text
拉取 origin → 合并队友分支到你分支 → 继续开发
4.4 功能完成后
text
切换到 main → 拉取 → 合并你的分支 → 推送

六、常见问题
6.1 看不到队友的分支
在终端中执行：

bash
git fetch origin
git branch -r
6.2 推送失败
先拉取再推送：

点击 拉取 origin

再点击 推送 origin

6.3 不小心在 main 上改了代码
点击 当前分支 → 新建分支

输入分支名（如 feature/紧急修复）

点击 发布分支

然后按正常流程合并回 main

6.4 想放弃本地所有修改
点击 存储库 → 放弃所有更改...

6.5 合并时出现冲突
按上面 3.11 解决冲突 的步骤操作即可。

七、一句话总结
你和队友各自在独立分支上开发，每天从 main 同步最新，完成后推送到各自分支，功能完成后合并到 main。需要队友代码时直接合并队友分支。
