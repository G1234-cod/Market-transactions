# 智能二手商品发布助手 - 开发任务分配与里程碑

## 里程碑一：环境基建与数据底层 (Day 1-2)

万丈高楼平地起，这个阶段不写核心业务逻辑，只打通底层基础设施。

- **任务 1：代码协同环境初始化**
  - 在 GitHub 创建私有仓库，建立 `frontend`、`backend`、`database` 三个根目录。
  - 两人统一使用 **GitHub Desktop** 进行代码的 Clone、Commit 和 Push，规避复杂的 Git 命令行冲突。
  - 在 `backend` 中创建 `.env` 文件（**注意：将其加入 `.gitignore`**），分别填入：
    - 阿里云百炼 API Key（Qwen-VL-Max 视觉识别）
    - DeepSeek 官方 API Key（DeepSeek-V4-Pro 文本生成）
- **任务 2：MySQL 数据库建表 (DDL)**
  - 执行 `database/schema.sql`，建立核心的**四张**表：`users`、`market_prices`、`published_items`、`ai_audit_logs`。
  - **关键动作：** 执行 `database/seed.sql`，往 `market_prices` 表灌入 20-30 条校园常见二手物品测试数据（如：罗技 G610 键盘、iPad Air 4 等），并设定均价/最高/最低参考价。
  - **关键动作：** 往 `users` 表手动 INSERT 测试用户，并初始化 PBKDF2 密码哈希。

## 里程碑二：后端 AI 链路与数据联动 (Day 3-5)

这个阶段是项目"最硬核"的后端攻坚，要求跑通"上传 → 识别 → 查库"的完整逻辑链条。

- **任务 1：搭建 FastAPI 网关与 LLM 客户端抽象层**
  - 搭建 FastAPI 基础框架，配置 CORS 中间件。
  - 创建 `backend/app/llm/` 模块，用策略模式封装 `BaseLLMClient` 抽象基类。
  - 分别实现 `QwenVLClient`（阿里云百炼）和 `DeepSeekClient`（DeepSeek 官方）两个子类。
  - 创建 `.env` 配置管理，分别存储两个平台的 API Key、Base URL 等。
- **任务 2：用户认证系统**
  - 实现 PBKDF2-SHA256 密码哈希与验证函数。
  - 编写 `POST /api/v1/register`（注册新用户，用户名唯一校验）。
  - 编写 `POST /api/v1/login`（用户名 + 密码登录鉴权）。
  - 前端：LoginPage.vue（登录页）+ RegisterPage.vue（注册页）+ Vue Router 路由守卫。
- **任务 3：视觉提取接口**
  - 编写 `POST /api/v1/extract` 接口：接收 FormData 图片流，保存到 `static/uploads/`。
  - 调用 Qwen-VL-Max，用 Few-shot + Schema 强约束 Prompt 返回结构化 JSON。
  - **容错设计：** Python 端用 `json.loads` + Pydantic 模型双重校验，失败返回友好错误。
- **任务 4：查价引擎与模糊匹配**
  - 编写 `price_service.py`：接收 `brand` + `model`，查询 `market_prices` 表。
  - **核心难点攻坚——三层匹配策略：**
    1. 精确匹配（`WHERE brand=X AND model=Y`）
    2. 未命中 → `difflib.SequenceMatcher` 计算相似度，取最高且 > 0.7 的
    3. 仍未命中 → 返回 `None`，标注"暂无市场行情数据"
- **任务 5：流式生成接口**
  - 编写 `POST /api/v1/generate` 接口（SSE）：接收用户确认后的物品信息 + 行情数据。
  - 组装 System Prompt（二手带货专家人设）+ User Prompt（物品特征 + 行情上下文）。
  - 调用 DeepSeek-V4-Pro 流式 API，用 FastAPI `StreamingResponse` 实时推送给前端。
- **任务 6：全链路串联测试**
  - 在终端打印完整日志：视觉识别 JSON → 查价结果 → 组装的 Prompt → SSE 流输出。
  - 确认各环节之间的数据结构正确传递，没有断层。

## 里程碑三：前端工程与大模型流式渲染 (Day 6-8)

这一步将后端的冰冷数据转化为极佳的 C 端用户交互体验。

- **任务 1：Vue 3 项目极速搭建**
  - 使用 Vite 创建 Vue 3 项目，集成 TailwindCSS、Vue Router。
  - 构建页面骨架：顶部导航栏（发布/历史/商城/用户信息）+ 路由守卫。
- **任务 2：图片上传与本地预处理**
  - 实现 `ImageUploader.vue`：支持拖拽上传和点击上传。
  - 浏览器端 Canvas 压缩（设定最大宽高 1920px，JPEG quality 0.8），降低后端传输压力。
  - 调用 `POST /api/v1/extract`，接收识别结果。
- **任务 3：确认中间页 + 输入校验**
  - 实现 `ConfirmCard.vue`：展示 AI 识别的品类/品牌/型号/成色，每项均可编辑修改。
  - **三级校验体系：** ok（绿框通过）/ warn（黄框提示但放行）/ error（红框拦截）。
  - 修改后重新请求查价接口，动态更新价格区间展示 + 行情价格条图。
  - 「💾 存草稿」按钮：未完成的信息可暂存为草稿，后续从历史页发布。
- **任务 4：SSE 流式生成对接与打字机渲染**
  - 前端封装 SSE 读取工具（基于 `fetch` + `ReadableStream`），解析 `data:` 行。
  - 实现 `TypewriterText.vue`：利用 Vue 3 `ref` 响应式变量，逐字追加渲染 + 光标闪烁。
  - 流结束后展示最终建议售价，提供「保存到发布记录」和「复制文案」按钮。
- **任务 5：发布历史页**
  - 实现 `HistoryPage.vue`：调用 `GET /api/v1/history` 获取当前用户的发布记录列表。
  - 卡片式展示：商品标题、文案摘要、建议价格、状态标签、日期。
  - 操作按钮：「下架」（published→delisted）、「发布」（draft→published）。
  - 统计概览面板（全部/已发布/已下架）+ 状态下拉筛选 + 关键词搜索框。

## 里程碑四：商城与用户生态 (Day 9-10)

行百里者半九十，最后一步是把零散的功能组合成一个完整的产品生态。

- **任务 1：二手商城模块**
  - 后端：`GET /api/v1/market`（JOIN users 获取卖家名，支持 keyword + category 筛选）。
  - 前端：`MarketPage.vue` — 栅格卡片展示所有已发布商品，分类下拉 + 关键词搜索，无需登录即可浏览。
- **任务 2：前后端联调整体测试**
  - 从前端浏览器真实上传一张旧物品的图片，静待几秒钟。
  - 验证能否顺利走完 **"登录 → 上传 → 识别 → 用户确认 → 流式生成文案 → 落库 → 商城展示"** 全流程。
- **任务 3：业务落库与安全审计记录**
  - 在生成流程结束后追加数据库操作：
    1. 将最终文案、定价及图片本地路径存入 `published_items`。
    2. 无论成功与否，将 API 耗时、模型名称、原生 JSON 报文、报错信息 INSERT 进 `ai_audit_logs`。
- **任务 4：工程文档沉淀（极其重要）**
  - 整理联调中遇到的坑（CORS 报错、模型 JSON 解析失败、型号匹配边界情况、密码哈希实现等）。
  - 将排查过程与解决方案记录到项目文档中，作为期末报告、软著申请与面试素材。
