- - # 智能二手商品发布助手 - 开发任务分配与里程碑
  
    ### 里程碑一：环境基建与数据底层 (建议耗时：Day 1-2)
  
    万丈高楼平地起，这个阶段不写核心业务逻辑，只打通底层基础设施。
  
    - **任务 1：代码协同环境初始化**
      - 在 GitHub 创建私有仓库，建立 `frontend` 和 `backend` 两个根目录。
      - 两人统一使用 **GitHub Desktop** 进行代码的 Clone、Commit 和 Push，彻底规避复杂的 Git 命令行冲突，保持本地与云端代码的丝滑同步。
      - 在 `backend` 中创建 `.env` 文件（**注意：将其加入 `.gitignore`**），各自填入申请好的多模态大模型 API Key（如 Qwen-VL 或 DeepSeek）。
    - **任务 2：MySQL 数据库建表 (DDL)**
      - 建立核心的**四张**表：用户表 (`users`)、行情基准表 (`market_prices`)、**商品发布记录表 (`published_items`)**、AI调用审计表 (**`ai_audit_logs`**)。
      - **关键动作：** 手动编写 SQL `INSERT` 语句，往 `market_prices` 表里灌入 20-30 条常见的校园二手物品测试数据（例如：罗技 G610 键盘、iPad Air 4 等），并设定好对应的最高/最低参考价。
  
    ### 里程碑二：后端 AI 链路与数据联动 (建议耗时：Day 3-5)
  
    这个阶段是项目“最硬核”的后端攻坚，要求跑通“上传 -> 识别 -> 查库”的完整逻辑链条。
  
    - **任务 1：构建 FastAPI 网关与视觉提取接口**
      - 搭建基础的 FastAPI 框架，编写 `/api/v1/extract` 接口接收前端传来的图片（FormData 流），并将图片保存在本地 `static/uploads/images` 目录下。
      - 对接视觉大模型 API，编写**结构化 Prompt**，强制模型输出只包含“品牌、型号、成色”的 JSON 字符串。
    - **任务 2：JSON 容错解析与 MySQL 模糊匹配**
      - 编写 Python 逻辑安全地解析大模型返回的 JSON。
      - **核心难点攻坚：** 拿着解析出来的“型号”，去 MySQL 的 `market_prices` 表中执行查询（使用 `LIKE` 模糊匹配或 Python 的 `difflib` 计算相似度），成功捞出对应的价格区间。
    - **任务 3：大模型 Prompt 拼装测试**
      - 在后端终端里打印测试：将“图片识别出的成色”+“数据库查出的价格”拼接成一段完整的后台日志，确认数据联动没有断层。
  
    ### 里程碑三：前端工程与大模型流式渲染 (建议耗时：Day 6-8)
  
    这一步将后端的冰冷数据转化为极佳的 C 端用户交互体验。
  
    - **任务 1：Vue 3 页面极速搭建**
      - 打开 Trae、Cursor 或 Windsurf 等 AI 编程工具，利用自然语言直接生成一个包含“拖拽上传区”和“右侧结果展示区”的现代化前端骨架。
      - 实现图片上传前的本地预览和压缩，降低后端网络传输压力。
    - **任务 2：Server-Sent Events (SSE) 流式接口对接**
      - 后端新增一个流式生成接口 `/api/v1/generate`，对接文本大模型（如 DeepSeek-V3）。
      - 配置 FastAPI 返回 `StreamingResponse`，大模型每吐出一个 Token，立刻推给前端。
    - **任务 3：前端打字机组件渲染**
      - 前端接收 SSE 流数据，利用 Vue 3 的响应式变量（ref/reactive），在页面上实现炫酷的“打字机”实时渲染效果，展示最终的带货文案和定价。
  
    ### 里程碑四：全链路闭环与工程文档沉淀 (建议耗时：Day 9-10)
  
    行百里者半九十，最后一步是把零散的功能组合成一个企业级的产品。
  
    - **任务 1：前后端联调整体测试**
      - 从前端浏览器真实上传一张旧键盘的图片，静待几秒钟，验证能否顺利走完“识别->查价->流式生成文案”的全流程。
    - **任务 2：业务落库与安全审计记录**
      - 在 FastAPI 的生成流程结束后追加最后两步数据库操作：
        1. 将生成的最终文案、定价以及**本地图片的相对物理路径**存入 **`published_items`** 表。
        2. 无论生成成功与否，将本次 API 的耗时、原生 JSON 报文及报错信息用 SQL `INSERT` 进 **`ai_audit_logs`** 表。
    - **任务 3：踩坑复盘与知识库归档（极其重要）**
      - 在联调过程中，必然会遇到诸如 FastAPI 跨域 (CORS) 报错、大模型 JSON 格式化失败等工程问题。
      - 将这些问题的排查过程与破局代码，整理沉淀到由 VitePress 驱动的个人静态知识库中，作为期末报告、软著申请与未来面试的顶级素材。
