# 智能二手商品发布助手 —— 项目完整文档


## 一、项目概述

### 1.1 项目目的

**一句话概括：让二手商品发布像拍照一样简单。**

用户只需要拍一张照片，系统自动完成：商品识别、瑕疵检测、文案生成、智能定价、一键发布。

### 1.2 解决的核心问题

| 问题 | 传统方式 | 本项目 |
|:---|:---|:---|
| 商品信息填写 | 手动打字，繁琐耗时 | AI 自动生成标题、描述 |
| 瑕疵描述 | 用户自己找、自己写 | AI 自动检测并画圈标注 |
| 定价 | 凭感觉，缺乏数据支撑 | AI 结合行情 + 瑕疵程度定价 |
| 找同款 | 文字搜索，不准确 | 以图搜图，拍照匹配相似商品 |
| 平台局限 | 闲鱼等平台不支持自动填信息 | 独立系统，全自动流程 |


### 1.3 目标用户

- 二手商品卖家（快速发布）
- 二手商品买家（浏览、搜索、比价）


## 二、技术选型

### 2.1 整体技术栈

| 层级 | 技术 | 用途 |
|:---|:---|:---|
| **后端框架** | FastAPI | Web API 服务 |
| **数据库** | MySQL | 业务数据存储 |
| **向量数据库** | Qdrant / FAISS | 以图搜图向量检索 |
| **深度学习框架** | PyTorch | 模型加载与推理 |
| **目标检测** | YOLOv8 | 物品识别、瑕疵检测 |
| **多模态模型** | CLIP | 以图搜图特征提取 |
| **大模型 API** | Qwen-VL | 图片理解、商品识别 |
| **大模型 API** | DeepSeek | 文案生成、智能定价 |
| **图像处理** | rembg + OpenCV | 去背景、去噪、瑕疵检测 |
| **前端框架** | Vue 3 | 用户界面 |
| **样式框架** | TailwindCSS | 界面样式 |
| **容器化** | Docker | Qdrant 部署 |

### 3.2 核心流程

```
用户上传图片
    ↓
图片预处理（尺寸统一 + 去背景 + 去噪）
    ↓
    ├─→ 物品识别（YOLO + Qwen 并行）
    │       ↓
    │   双模型比对 → 不一致 → 保存错误数据
    │
    ├─→ 瑕疵检测（YOLO / OpenCV）
    │       ↓
    │   画圈标注（不同形状+颜色代表不同程度）
    │       ↓
    │   DeepSeek 定价（结合瑕疵程度 + 市场行情）
    │
    └─→ 以图搜图（CLIP 特征提取 + Qdrant 检索）
    
    ↓
自动填充标题 + 描述 + 价格 + 瑕疵标注图
    ↓
用户确认 → 一键发布到商城
```
