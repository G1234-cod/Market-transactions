# 智能二手商品发布助手 — AI 模型原理详解

---

## 👁️ YOLO：目标检测的“火眼金睛”

### 1. 它要解决什么问题？

给你一张图片，YOLO要回答两个问题：
- **“里面有什么？”**（分类）
- **“分别在哪儿？”**（定位）

传统方法分两步走：先猜一千个可能的位置，再一个个判断是什么。YOLO的革命性在于——**只看一次，全部说完**。

### 2. 它的核心思想

YOLO把一张图片想象成一个棋盘，把它分成 `S×S` 个格子（比如 `7×7 = 49` 个格子）。

- **规则**：每个格子只负责预测**中心点落在它里面的物体**。
- **输出**：每个格子输出一个固定长度的向量，包含：
  - 这个格子里有没有物体？（置信度）
  - 如果有，物体的边界框相对于格子的偏移量（x, y, w, h）
  - 如果有，它属于哪个类别？（如手机、划痕）

**一次前向传播**，49个格子同时输出结果，所以叫“You Only Look Once”——只看一次。

### 3. 内部是怎么一步步处理的？

**第一步：提取特征（骨干网络）**

图片输入后，先经过一系列卷积层（CSPDarknet53 + C2f模块），像筛子一样层层过滤，把图片压缩成一张“特征图”。这张图比原图小很多，但每个像素点都浓缩了周围一大片区域的信息。

**第二步：融合不同尺度的特征（颈部网络）**

YOLO要同时检测大物体（如整个手机）和小物体（如屏幕上的细微划痕）。

- 浅层特征图：分辨率高，包含丰富的**位置细节**，适合找小东西。
- 深层特征图：分辨率低，包含抽象的**语义信息**，适合判断是什么东西。

YOLO通过**PAN-FPN结构**，把浅层和深层的特征图融合在一起，让每一层都既有位置信息又有语义信息。

**第三步：做出预测（检测头）**

YOLOv8采用**无锚框设计**。它不再预设一堆固定大小的“锚框”，而是**直接预测物体的中心点和宽高**。这让模型更灵活，训练时也少了很多需要手动调整的参数。

**第四步：筛选结果（非极大值抑制）**

同一个物体可能会被多个格子同时检测到，产生一堆重叠的框。YOLO通过**非极大值抑制（NMS）** 算法，只保留置信度最高的那个框，把重复的删掉。

### 4. 它是怎么训练的？（有监督学习）

**第一步：准备数据**

你需要收集数千张图片，然后用标注工具（如LabelImg）人工框出每个物体的位置，并写上类别名称。每张图片对应一个TXT文件，每行记录一个物体的信息：`类别ID x_center y_center width height`（所有坐标都是归一化后的比例值）。

**第二步：加载预训练权重**

YOLO官方提供了在COCO数据集（80个常见类别）上预训练好的权重。这个模型已经能识别狗、猫、汽车等常见物体，但不知道“划痕”、“磕碰”是什么。所以我们需要在它的基础上继续学习。

**第三步：微调训练**

把预训练权重加载进来，然后用你自己的标注数据继续训练。这个过程叫“微调”（Fine-tuning）。模型会调整自己的内部参数，让自己越来越擅长识别你特有的类别。

**第四步：计算损失，反向传播**

每次训练时，模型会做一个预测，然后把预测结果和真实标注进行对比，计算出**损失值**：

- **分类损失**：预测的类别对不对？
- **回归损失**：预测的框位置准不准？（YOLOv8使用CIoU Loss，综合考虑了框的重叠面积、中心点距离和长宽比）

然后通过反向传播，把损失值一层层传回去，调整每个神经元的权重，让下一次预测更准。

**第五步：反复迭代**

这个过程要重复成千上万次（称为“Epochs”），每次用不同的图片。最终模型会收敛，输出一个 `best.pt` 权重文件。

**第六步：推理使用**

训练完成后，加载 `best.pt`，输入一张新图片，模型直接输出检测结果（框 + 类别 + 置信度）。这个过程只需要几十毫秒，所以能做到“实时”。

## 🧠 CLIP：多模态理解的“桥梁”

### 1. 它要解决什么问题？

YOLO能告诉你“这是什么物体”，但它不理解“语义”——它不知道“一只狗在草地上玩”和一张图片是不是匹配的。

CLIP要解决的核心问题是：**如何让计算机同时理解图片和文字，并知道它们之间的对应关系。**

### 2. 它的核心思想

CLIP构建了一个**共享的“语义空间”**。

- 每一张图片，经过编码器，变成这个空间里的一个点（向量）。
- 每一段文字，经过编码器，也变成这个空间里的一个点（向量）。
- **训练目标**：让匹配的“图片-文字对”在这个空间里距离很近，不匹配的距离很远。

想象一个巨大的图书馆，每本书都放在一个高维空间里。内容相似的书（比如不同版本的水果百科全书）放在同一个区域；内容不同的书（比如讲水果的和讲汽车的）放在不同的区域。

### 3. 内部架构：双塔结构

CLIP采用**双编码器架构**，两个编码器独立工作，最后在同一个空间里对齐：

- **图像编码器**：负责把图片变成向量。早期版本用ResNet，新版本用Vision Transformer（ViT）。ViT把图片切成16×16的小块，像处理文字一样处理图像。
- **文本编码器**：负责把文字变成向量。用的是Transformer架构，和GPT类似。

两个编码器输出的向量**维度相同**（比如都是512维），所以可以在同一个空间里比较。

### 4. 它是怎么训练的？（自监督学习，不需要人工标注）

**第一步：收集训练数据**

CLIP的训练数据是**从互联网上自动爬取的**——总共4亿对（图片, 文字描述）。这些数据本身就是天然标注：一张图片配一段文字描述，不需要任何人手动去分类。

**第二步：对比学习——核心训练策略**

这是CLIP最精彩的部分。假设一个训练批次里有N张图片和对应的N段文字描述。

1.  CLIP把N张图片全部编码成向量：`I₁, I₂, ..., Iₙ`
2.  CLIP把N段文字全部编码成向量：`T₁, T₂, ..., Tₙ`
3.  计算一个 `N×N` 的相似度矩阵。位置 `(i, j)` 表示第i张图片和第j段文字的相似度（用余弦相似度）。
4.  **关键**：在矩阵中，对角线上的 `(i, i)` 是匹配的图文对（正样本），其他所有 `(i, j)` 都是不匹配的（负样本）。
5.  训练目标是**让对角线上的值尽可能大，非对角线上的值尽可能小**。

这就是**对比损失（Contrastive Loss）**。模型通过这种方式，学会了把匹配的图文对拉近，不匹配的推远。这个过程不需要任何人工标注。

**第三步：零样本泛化**

经过4亿图文对的训练，CLIP具备了强大的**零样本（Zero-shot）能力**。即使给一张它从未见过的图片，它也能通过语义空间判断出“这段文字描述的是这张图吗”。

### 5. 在项目中的应用：特征提取器

我们**直接使用CLIP预训练好的图像编码器**，不做任何额外训练。流程如下：

1.  用户上传一张商品图片。
2.  后端调用CLIP的图像编码器，把图片变成512维的特征向量。
3.  这个向量代表了图片的“语义指纹”——它捕捉了图片的风格、内容、颜色分布等深层特征。
4.  这个向量被存入Qdrant，用于后续的以图搜图。

我们之所以不训练CLIP，是因为：
- 训练CLIP需要超大规模数据（4亿图文对）和巨额算力（数百块GPU训练数周）。
- 预训练模型已经足够强大，我们的数据量（几千张）对它来说只是“九牛一毛”，训练反而可能破坏它已有的能力。

## 🔍 Qdrant：向量检索的“搜索引擎”

### 1. 它要解决什么问题？

当系统中有了成千上万张商品图片，用户上传一张新图片想找相似款时，怎么办？

- 用MySQL：只能比对文字标签（如“iPhone”），找不到视觉上相似但标签不同的图片。
- 暴力搜索：把新图片和所有图片逐一比对，O(N)的时间复杂度，N=100万时根本不可行。

Qdrant要解决的核心问题是：**如何在海量向量中，毫秒级找到最相似的K个？**

### 2. 它的核心原理：向量相似度搜索

Qdrant存储的数据是**高维向量**（来自CLIP提取的特征）。它计算两个向量之间的距离（如余弦相似度），距离越近，表示在语义上越相似。

- **余弦相似度**：计算两个向量之间的夹角余弦值。值越接近1，表示方向越一致（语义越相似）。
- **欧氏距离**：计算两个向量之间的直线距离。值越小，表示在空间里越接近。

搜索时，用户图片变成查询向量，Qdrant在数据库中找出与该向量最相似的Top-K个向量，返回对应的商品ID。

### 3. 内部实现：HNSW索引算法

如果Qdrant把所有向量逐一对比，100万 × 100万 = 太慢。所以它采用**近似最近邻（ANN）搜索**——不找最精确的，而是找一个“足够好”的，速度能快上千倍。

最核心的算法是**HNSW（Hierarchical Navigable Small World）**——分层可导航小世界图。

**想象一个场景：**

你要在一个巨大的城市里找一家最好吃的面馆，怎么办？

- 你不会逐条街道去问——太慢。
- 你会先打开手机地图（**顶层图**），看看面馆在哪个大区（如“海淀区”）。
- 进入海淀区后（**中层图**），找到更具体的街道（如“中关村大街”）。
- 最后在街道上（**底层图**），精确找到那家面馆。

**HNSW就是这么干的：**

- **顶层图**：节点很少（比如只有几十个），它们之间的距离很远。搜索时，你从任何一个节点出发，沿着最接近目标的路径跳几次，就能定位到大致区域。复杂度 O(log N)。
- **中层图**：节点增多，距离变近。在第一步锁定的区域里，进一步细找。
- **底层图**：节点最多，距离最近。在这一层做精确搜索，找到最终的K个结果。

**插入新向量时**：HNSW会判断这个新向量应该连接哪些节点（类似于朋友圈）。连接的原则是：**在图中保持“小世界”特性**——任何两个节点之间，都可以通过少量跳转到达。

### 4. 在项目中的应用：数据索引 + 搜索

**Qdrant不需要训练**。它只是一个存储和检索的工具。我们对它做的是“数据索引”：

**第一步：插入数据（索引）**

1.  每发布一个商品，系统会调用CLIP模型提取图片的512维特征向量。
2.  把这个向量和商品ID、图片URL等元数据组合成一个“点（Point）”，插入到Qdrant的一个集合（Collection）里。
3.  Qdrant自动更新HNSW索引，把新点加入图结构中。

**第二步：执行搜索**

1.  用户上传一张图片用于搜索。
2.  后端调用CLIP提取查询图片的特征向量。
3.  把这个向量发送给Qdrant，附上参数 `top_k=10`。
4.  Qdrant利用HNSW索引，快速找出与查询向量最相似的10个点，返回它们的商品ID和相似度分数。
5.  后端拿着商品ID去MySQL查询具体信息（标题、价格、图片URL等），返回给前端展示。

### 5. 索引维护

当商品被下架或删除时，需要从Qdrant中删除对应的点，避免返回已下架的商品。

项目中的 `scripts/index_items.py --all` 脚本，就是遍历所有已发布商品，批量提取特征并插入Qdrant的维护工具。

## 💎 三者如何协作？

| 阶段 | 哪个组件 | 做什么 | 什么学习方式 |
|:---|:---|:---|:---|
| **训练阶段** | YOLO | 学习“物体是什么、在哪里” | **有监督**（人工标注数据） |
| **训练阶段** | CLIP | 学习“图文匹配关系” | **自监督**（互联网图文对） |
| **训练阶段** | Qdrant | 不学习，只是存储和索引 | **无监督**（只是工具） |
| **推理阶段** | YOLO | 检测新图片中的物体和瑕疵 | 调用训练好的模型 |
| **推理阶段** | CLIP | 提取新图片的特征向量 | 调用预训练模型 |
| **推理阶段** | Qdrant | 在海量向量中找相似的 | 执行索引搜索 |

**完整流程：**

1.  用户上传图片 → YOLO检测出物品和瑕疵位置 → 画圈标注 → 前端展示给用户看
2.  同一张图片 → CLIP提取特征向量 → Qdrant检索相似商品 → 返回“相似推荐”
3.  用户确认商品信息 → DeepSeek生成文案 → 发布 → 图片向量自动存入Qdrant索引
