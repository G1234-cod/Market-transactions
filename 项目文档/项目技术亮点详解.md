## 1. AI 架构设计

### 亮点描述

系统采用 **"视觉识别 + 目标检测 + 语义理解 + 文本生成"** 四层 AI 架构，各模型各司其职又相互配合。

### 技术实现

| 模型 | 职责 | 选型理由 |
|:---|:---|:---|
| **Qwen-VL-Max** | 视觉理解、结构化识别 | 中文场景最优，JSON 输出稳定 |
| **YOLOv8** | 物品检测、瑕疵定位 | 实时性好，易训练部署 |
| **CLIP** | 多模态特征提取 | 以图搜图/以文搜图核心 |
| **DeepSeek** | 文案生成、智能定价 | 中文带货文案质量顶级 |
| **rembg** | 图像去背景 | 轻量，专注主体提取 |

### 技术价值

- 各模型独立部署，可单独升级替换
- 通过策略模式统一管理 LLM 客户端
- 形成完整的 AI 能力矩阵


## 2. 双模型并行验证

### 亮点描述

YOLO（本地检测）和 Qwen-VL（云端识别）同时处理同一张图片，结果比对不一致时自动记录到错题本，形成数据闭环。

### 技术实现

```
用户上传图片
    ↓
┌─────────────────┴─────────────────┐
│                                   │
YOLO 本地检测                    Qwen-VL 云端识别
(毫秒级，80类COCO)              (秒级，语义理解)
│                                   │
└─────────────────┬─────────────────┘
                  ↓
            结果比对引擎
                  ↓
         ┌───────┴───────┐
         │               │
      一致            不一致
         │               │
         ↓               ↓
     正常返回      收集到 hard_cases
                  (图片 + 错误标签 + 正确标签)
```

### 技术价值

- **互补性**：YOLO 定位准但类别有限，Qwen 理解强但成本高
- **自监督**：不一致数据自动标注，无需人工干预
- **持续进化**：错误数据积累越多，模型迭代越有效

## 4. 以图搜图系统

### 亮点描述

基于 CLIP 多模态模型 + Qdrant 向量数据库，同时支持以图搜图和以文搜图。

### 技术实现

```
以图搜图流程：
用户上传图片
    ↓
CLIP 图像编码器 → 512维特征向量
    ↓
Qdrant 向量检索 (COSINE 相似度)
    ↓
返回 Top-K 商品ID
    ↓
MySQL 查询商品详情
    ↓
返回相似商品列表 + 相似度分数

以文搜图流程：
用户输入文字
    ↓
CLIP 文本编码器 → 512维特征向量
    ↓
(同以上流程)
```

### 技术架构

| 组件 | 技术 | 说明 |
|:---|:---|:---|
| 特征提取 | open-clip-torch (ViT-B-32) | 512维向量，图文统一空间 |
| 向量数据库 | Qdrant (Docker) | HNSW 索引，毫秒级检索 |
| 部署方式 | Docker 容器 | 一键启动，数据持久化 |
| 索引维护 | 自动 + 手动 | 发布时自动索引，脚本批量索引 |

### 技术价值

- **统一语义空间**：图片和文字在同一空间比较
- **毫秒级检索**：HNSW 算法支持百万级数据
- **零样本能力**：无需训练即可支持任意文字描述

## 6. 瑕疵检测与视觉编码

### 亮点描述

YOLO 检测瑕疵后，使用**不同形状 + 不同颜色**代表**不同程度的损伤**，实现直观的视觉编码。

### 程度分级系统

| 形状 | 颜色 | 十六进制 | 程度 | 代表含义 |
|:---|:---|:---|:---|:---|
| 圆形 | 红色 | #FF0000 | 重度 | 影响功能/结构 |
| 矩形 | 橙色 | #FF8C00 | 中度 | 影响外观 |
| 多边形 | 黄色 | #FFD700 | 轻度 | 细微痕迹 |
| 虚线框 | 蓝色 | #00BFFF | 轻微 | 表面污渍 |

### 技术实现

```python
# 程度分级逻辑
def _determine_severity(defect_type, area_ratio, bbox):
    # 裂痕 → 重度
    if defect_type == 'crack':
        return 'severe'
    
    # 磕碰：面积大 → 中度，面积小 → 轻度
    if defect_type == 'dent':
        if area_ratio > 0.001:
            return 'moderate'
        return 'minor'
    
    # 划痕：细长且面积大 → 中度，否则轻度
    if defect_type == 'scratch':
        aspect_ratio = max(width, height) / (min(width, height) + 1)
        if aspect_ratio > 5 and area_ratio > 0.0005:
            return 'moderate'
        return 'minor'
    
    # 污渍 → 轻微
    if defect_type == 'stain':
        return 'slight'
```

### 技术价值

- **直观性**：用户一眼就能看懂瑕疵位置和严重程度
- **信息隐层**：程度信息不直接展示文字，避免争议
- **标注效率**：无需人工圈画，全自动标注


## 7. LLM 策略模式抽象

### 亮点描述

所有 LLM 客户端通过抽象基类统一管理，DeepSeek/Qwen 可无缝切换，未来接入 GPT/Claude 只需新增子类。

### 技术实现

```python
# 抽象基类
class BaseLLMClient(ABC):
    @abstractmethod
    async def chat(self, messages, **kwargs) -> str: ...
    
    @abstractmethod
    async def chat_stream(self, messages, **kwargs) -> AsyncIterator[str]: ...

# 实现类
class DeepSeekClient(BaseLLMClient): ...
class QwenVLClient(BaseLLMClient): ...
class OpenAIClient(BaseLLMClient): ...  # 未来扩展

# 工厂模式
class LLMClientFactory:
    @classmethod
    def create_client(cls, client_type, config) -> BaseLLMClient:
        ...
```

### 技术价值

- **开闭原则**：对扩展开放，对修改封闭
- **模型解耦**：业务代码不依赖具体模型实现
- **切换灵活**：一键切换模型供应商

## 9. 人机协作确认页

### 亮点描述

AI 识别后不直接生成文案，而是先展示确认页，用户可修改识别结果，形成人机协作的发布流程。

### 三级校验体系

| 级别 | 颜色 | 条件 | 行为 |
|:---:|:---|:---|:---|
| **ok** | 🟢 绿框 | 字段完整且合理 | 通过，允许发布 |
| **warn** | 🟡 黄框 | 字段可疑但允许继续 | 提示但允许继续 |
| **error** | 🔴 红框 | 字段为空或明显错误 | 拦截，阻止发布 |

### 技术实现

```javascript
// 前端校验逻辑
function validateField(value, fieldName) {
    if (!value || value.trim() === '') {
        return { level: 'error', message: `${fieldName}不能为空` }
    }
    if (isSuspicious(value)) {
        return { level: 'warn', message: `${fieldName}可能不准确，请确认` }
    }
    return { level: 'ok', message: '' }
}
```

### 技术价值

- **提升可用性**：避免 LLM 识别错误导致文案不可用
- **用户参与感**：用户是最终决策者
- **错误容错**：识别不准时用户可纠正


## 10. SSE 流式体验

### 亮点描述

文案生成使用 Server-Sent Events (SSE) 协议，实现打字机效果，用户实时看到生成过程。

### 事件类型

| 事件 | 说明 | 触发时机 |
|:---|:---|:---|
| `start` | 开始生成 | 请求开始 |
| `content` | 内容片段 | 每个 token 生成时 |
| `done` | 生成完成 | 正常结束 |
| `error` | 生成异常 | 异常发生时 |

### 前端处理

```javascript
// SSE 事件处理
switch (data.type) {
    case 'start':
        loading.value = true
        break
    case 'content':
        generatedText.value += data.content  // 打字机效果
        break
    case 'done':
        loading.value = false
        break
    case 'error':
        error.value = data.message
        loading.value = false
        break
}
```

### 技术优势

| 对比 | 传统 Polling | WebSocket | SSE |
|:---|:---|:---|:---|
| 单向/双向 | 双向 | 双向 | 单向（服务器→客户端） |
| 自动重连 | ❌ | ❌ | ✅ |
| 协议 | HTTP | WS | HTTP/2 |
| 实现复杂度 | 低 | 高 | 低 |
| 适用场景 | 实时双向通信 | 实时双向通信 | 服务器推送（文案生成） |

### 技术价值

- **实时反馈**：用户无需等待完整响应
- **体验好**：打字机效果模拟真人打字
- **简单可靠**：SSE 协议轻量，自动重连


## 11. 三层查价策略

### 亮点描述

价格查询采用三层匹配策略，最大化行情数据命中率。

### 匹配策略

```
用户请求: brand="Apple", model="iPhone 13"

第1层：精确匹配
    SELECT ... WHERE brand='Apple' AND model='iPhone 13'
    ✅ 命中 → 返回价格
    ❌ 未命中 → 进入第2层

第2层：模糊匹配 (difflib)
    获取该品牌所有型号
    计算模型相似度 (SequenceMatcher)
    取相似度 > 0.7 的最高分
    ✅ 命中 → 返回价格
    ❌ 未命中 → 进入第3层

第3层：返回空
    标注"暂无市场行情数据"
```

### 技术实现

```python
def query_price(brand: str, model: str):
    # 第1层：精确匹配
    row = await exact_match(brand, model)
    if row:
        return row
    
    # 第2层：模糊匹配
    rows = await get_by_brand(brand)
    best = max(rows, key=lambda r: similarity(model, r['model']))
    if best and similarity > 0.7:
        return best
    
    # 第3层：返回空
    return None
```

### 技术价值

- **高命中率**：模糊匹配覆盖拼写错误、型号变体
- **用户友好**：即使无精确匹配也能提供参考
- **可扩展**：后续可接入外部 API 作为第4层

## 13. 图像预处理管道

### 亮点描述

用户上传图片后，经过完整的预处理管道，提升 AI 识别准确率。

### 预处理流程

```
原始图片
    ↓
尺寸统一 (448×448)
    ↓
去背景 (rembg) → 聚焦主体
    ↓
去噪 (OpenCV 高斯滤波) → 减少噪点
    ↓
多物体判断 → 选占比最大的物体
    ↓
预处理完成图片
    ↓
送入 YOLO + Qwen 双模型
```

### 技术实现

```python
# 预处理管道
def process(image):
    # 1. 尺寸统一
    image = image.resize((448, 448))
    
    # 2. 去背景
    image = rembg.remove(image)
    
    # 3. 去噪
    image = cv2.GaussianBlur(image, (5, 5), 0)
    
    # 4. 多物体判断
    main_object = select_main_object(image)
    
    return main_object
```

### 技术价值

- **提升准确率**：去除背景干扰，聚焦主体
- **统一输入**：各模型输入尺寸一致
- **减少误检**：去噪后边缘更清晰
