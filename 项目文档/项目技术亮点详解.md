
## 一、双模型并行架构


### 1.1 架构设计

```
用户上传图片
    ↓
┌─────────────────────────────────────────────────────────────┐
│                    并行调用                                 │
│  ┌─────────────────┐    ┌─────────────────┐                │
│  │   YOLOv8        │    │   Qwen-VL       │                │
│  │  本地模型       │    │   云端API       │                │
│  │  快速推理       │    │  高精度识别     │                │
│  └─────────────────┘    └─────────────────┘                │
│         ↓                      ↓                          │
│    结果 A                   结果 B                        │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                      比对模块                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  结果一致 → 采用该结果                             │   │
│  │  结果不一致 → 以 Qwen 为准 + 保存错误数据          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 技术优势

| 优势 | 说明 |
|:---|:---|
| **高可用性** | 任一模型不可用时，另一个仍可工作 |
| **成本优化** | 逐步用本地模型替代 API，降低费用 |
| **数据飞轮** | 不一致的数据自动收集，用于模型迭代 |
| **精度验证** | 交叉验证，发现模型偏差 |

### 1.3 实现代码

```python
# 并行调用
import asyncio

async def recognize_with_two_models(image):
    yolo_task = asyncio.create_call_yolo(image))
    qwen_task = asyncio.create_call_qwen(image))
    yolo_result, qwen_result = await asyncio.gather(yolo_task, qwen_task)
    return compare_results(yolo_result, qwen_result)
```


## 二、瑕疵检测与程度分级


### 2.1 检测流程

```
预处理后图片
    ↓
YOLO / OpenCV 瑕疵检测
    ↓
┌─────────────────────────────────────────────────────────────┐
│  检测结果                                                  │
│  ├── 瑕疵1: 划痕 (位置: 120, 340, 180, 400)              │
│  ├── 瑕疵2: 磕碰 (位置: 200, 150, 260, 220)              │
│  └── 瑕疵3: 污渍 (位置: 50, 480, 130, 520)               │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  程度分级                                                  │
│  ├── 划痕: 面积 0.0012 → 轻度 → 黄色多边形               │
│  ├── 磕碰: 面积 0.0035 → 中度 → 橙色矩形                 │
│  └── 污渍: 面积 0.0008 → 轻微 → 蓝色虚线框               │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 形状与颜色分级系统

| 形状 | 颜色 | 程度 | 代表含义 |
|:---|:---|:---|:---|
| 圆形 | 红色 | 重度 | 影响功能，严重损伤 |
| 矩形 | 橙色 | 中度 | 影响外观，明显损伤 |
| 多边形 | 黄色 | 轻度 | 细微痕迹，轻微损伤 |
| 虚线框 | 蓝色 | 轻微 | 表面污渍，可清理 |

## 三、以图搜图（CBIR）


### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    CLIP 多模态模型                          │
│  ┌─────────────┐    ┌─────────────┐                       │
│  │  图片编码器  │    │  文本编码器  │                       │
│  │  512维向量   │    │  512维向量   │                       │
│  └─────────────┘    └─────────────┘                       │
│         ↓                  ↓                              │
│  ┌─────────────────────────────────────┐                   │
│  │     同一向量空间（512维）           │                   │
│  │  图片向量 ≈ 文本向量（语义对齐）    │                   │
│  └─────────────────────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│                    Qdrant / FAISS 向量库                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  item_id │ 特征向量（512维）    │  payload         │   │
│  │  10001   │ [0.123, 0.456, ...] │  {title, price}  │   │
│  │  10002   │ [0.234, 0.567, ...] │  {title, price}  │   │
│  │  ...     │  ...                │  ...             │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心功能

| 功能 | 输入 | 输出 | 原理 |
|:---|:---|:---|:---|
| 以图搜图 | 图片 | 相似商品列表 | 图片→特征向量→检索 |
| 以文搜图 | 文字描述 | 匹配商品列表 | 文字→特征向量→检索 |

```
│  └─────────────────────────────────────────────────────────────────┘      │
│                              ↓                                          │

│                              ↓                                          │
│  ┌─────────────────────────────────────────────────────────────────┐      │

│  │  ├── 本周错误数据 + 历史抽样数据                               │      │
│  │  ├── 数据增强 + 正则化防过拟合                                │      │
│  │  └── 微调模型 → 更新 best.pt                                  │      │
│  └─────────────────────────────────────────────────────────────────┘      │
│                              ↓                                          │
│  ┌─────────────────────────────────────────────────────────────────┐      │
│  │  模型更新                                                      │      │
│  │  └── 新模型部署上线 → 推理精度提升                            │      │
│  └─────────────────────────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 过拟合防护

| 防护措施 | 具体做法 |
|:---|:---|
| **数据混合** | 本周错误数据 + 历史数据（1:1） |
| **早停法** | 验证集连续 N 轮不提升则停止 |
| **数据增强** | 旋转、裁剪、翻转 |
| **正则化** | Dropout、Weight Decay |
| **学习率调度** | 余弦退火或阶梯式降低 |


## 五、图像预处理管道


### 5.1 处理流程

```
原始图片（用户上传）
    ↓
┌─────────────────────────────────────────────────────────────┐
│  ① 尺寸统一（448×448）                                    │
│  优点：统一输入尺寸，减少计算量                            │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  ② 多物体判断                                              │
│  ├── 轮廓检测 → 识别各个物体                              │
│  ├── 计算像素占比 → 选择主物体（>60%）                    │
│  └── 占比接近（50% vs 50%）→ 提示用户重新上传              │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  ③ 去背景（rembg）                                         │
│  └── 基于 U²-Net 的深度学习模型，精确分离商品主体          │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  ④ 去噪（高斯滤波）                                        │
│  └── 清除图片中的杂点、纹理干扰                            │
└─────────────────────────────────────────────────────────────┘
    ↓
预处理完成 → 用于后续识别和特征提取
```

### 5.2 技术选型

| 处理步骤 | 技术 | 优势 |
|:---|:---|:---|
| 尺寸统一 | Pillow resize | 高效、跨平台 |
| 多物体判断 | OpenCV 轮廓检测 | 实时、无需训练 |
| 去背景 | rembg（U²-Net） | 高精度、支持 CPU |
| 去噪 | 高斯滤波 | 简单有效、保留边缘 |
