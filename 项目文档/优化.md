## 二、改动一：自建模型逐步替代 Qwen

### 2.1 双模型并行架构

- 自行构建图像识别模型（YOLOv8），与 Qwen **并行运行**。
- 每次用户上传图片后，**本地模型结果与 Qwen 结果同时输出**，并进行比对。

### 2.2 错误数据收集机制

当本地模型与 Qwen 结果不一致时，判定为「错误图片」，执行以下保存操作：

**（1）图片保存**
- 将错误图片保存到 `data/error_data/images/`
- 命名规则：`item_{item_id}_{timestamp}.jpg`

**（2）标签文件保存**
- 为每张错误图片生成对应的 TXT 文件到 `data/error_data/labels/`
- TXT 文件内容格式：
  ```
  图片名称：item_001_20260623.jpg
  错误分类（本地模型输出）：鼠标
  正确分类（Qwen 输出）：鼠标垫
  比对时间：2026-06-23 14:30:25
  用户ID：xxx
  商品ID：xxx
  ```

**文件结构**：
```
data/error_data/
├── images/
│   ├── item_001_20260623.jpg
│   └── ...
├── labels/
│   ├── item_001_20260623.txt
│   └── ...
└── metadata/
    └── error_summary_2026W26.json   # 每周汇总统计
```

### 2.3 每周迭代训练

**流程**：
1. 以**每周**为周期触发训练任务。
2. 读取本周收集的所有错误数据（图片 + 正确标签）。
3. 调用本地模型进行**微调训练（Fine-tuning）**。
4. 训练完成后，更新本地模型权重。

**训练集构成**：
```
本周错误数据：200 张
历史抽样数据：200 张（从过去 4 周中各抽 50 张）
验证集：50 张（从上述混合数据中划分）
─────────────────────────────
总计训练样本：400 张
```

**过拟合防范措施**：

| 防范措施 | 具体做法 |
|----------|----------|
| **数据混合** | 每次训练时混合历史错误数据（1:1 比例） |
| **早停法** | 验证集准确率连续 N 轮不提升时停止 |
| **数据增强** | 旋转、裁剪、翻转等操作扩充样本 |
| **正则化** | Dropout、Weight Decay |
| **学习率调度** | 余弦退火降低学习率 |

### 2.4 错误数据存储

| 存储位置 | 内容 | 格式 |
|----------|------|------|
| `data/error_data/images/` | 错误图片 | JPG |
| `data/error_data/labels/` | 错误标签 | TXT |
| `hard_cases` 表 | 错误记录 | MySQL |


## 三、改动二：引入瑕疵检测与画圈标注

### 3.1 瑕疵检测模型

- 使用 **YOLOv8** 进行瑕疵检测，在商品图上进行**画框标注**。
- 支持的瑕疵类型：
  - 划痕（scratch）
  - 磕碰（dent）
  - 污渍（stain）
  - 裂痕（crack）
  - 掉漆（peeling）
  - 变形（deformation）

### 3.2 程度分级与视觉编码

**不同形状 + 不同颜色 = 不同程度**：

| 形状 | 颜色 | 程度 | 含义 |
|------|------|------|------|
| 圆形 | 红色 #FF0000 | 重度 | 影响功能/结构 |
| 矩形 | 橙色 #FF8C00 | 中度 | 影响外观 |
| 多边形 | 黄色 #FFD700 | 轻度 | 细微痕迹 |
| 虚线框 | 蓝色 #00BFFF | 轻微 | 表面污渍 |

> ⚠️ **程度信息不向用户披露**，仅供内部定价使用。

### 3.3 输出格式

```
检测结果：
- 商品名称：iPhone 14 Pro
- 瑕疵总数：3 处
  - 瑕疵1：位置 (120, 340, 180, 400)，类型：划痕，程度：轻度
  - 瑕疵2：位置 (200, 150, 260, 220)，类型：磕碰，程度：中度
  - 瑕疵3：位置 (50, 480, 130, 520)，类型：掉漆，程度：轻度
→ 发送给 DeepSeek 进行定价评估
```


## 四、改动三：增加图像预处理流程

### 4.1 预处理流程

```
原始图片（用户上传）
       ↓
┌──────────────────┐
│ 尺寸统一         │ → 固定 448×448
└──────────────────┘
       ↓
┌──────────────────┐
│ 多物体判断       │ → 选占比最大的物体
└──────────────────┘
       ↓
┌──────────────────┐
│ 去背景（rembg）   │ → 透明背景商品图
└──────────────────┘
       ↓
┌──────────────────┐
│ 去噪             │ → 高斯滤波
└──────────────────┘
       ↓
┌──────────────────┐
│ 预处理完成       │ → 用于后续识别和特征提取
└──────────────────┘
```

### 4.2 多物体处理策略

- 使用轮廓检测识别各个物体
- 计算每个物体的像素占比
- 选择**占比最大（> 60%）** 的物体作为主要识别目标
- 其他物体视为背景干扰，一并去除

### 4.3 原始图片保留

- 本地数据库**始终保存原始图片**（`static/uploads/raw/`）
- 预处理后的图片仅用于识别、检索等分析环节


## 五、改动四：增加以图搜图能力

### 5.1 技术架构

| 组件 | 技术 | 说明 |
|------|------|------|
| 特征提取 | CLIP (ViT-B/32) | 512 维特征向量 |
| 向量数据库 | Qdrant | Docker 部署，HNSW 索引 |
| 索引维护 | 自动 + 手动 | 发布时自动索引，脚本批量索引 |

### 5.2 检索流程

```
用户上传图片
    ↓
CLIP 提取特征向量 (512维)
    ↓
Qdrant 向量检索 (COSINE 相似度)
    ↓
返回 Top-K 商品ID
    ↓
MySQL 查询商品详情
    ↓
返回相似商品列表 + 相似度分数
```

### 5.3 索引更新时机

| 时机 | 操作 |
|------|------|
| 新商品上架 | 自动提取特征 → 加入 Qdrant |
| 商品下架 | 从 Qdrant 中删除 |
| 批量索引 | `python scripts/index_items.py --all` |


## 六、完整数据流转图

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                              用户上传商品图片                                           │
│                              （原始图片永久保存）                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘
                                          ↓
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                    ① 图片预处理（改动三）                                               │
│  ├── 尺寸统一（固定 448×448）                                                         │
│  ├── 多物体判断 → 选占比最大的物体                                                    │
│  ├── 去背景（rembg）                                                                  │
│  └── 去噪（高斯滤波）                                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘
                                          ↓
                    ┌─────────────────────┴─────────────────────┐
                    ↓                                           ↓
┌───────────────────────────────────────────┐   ┌───────────────────────────────────────────┐
│  ② 识别分类（改动一）                     │   │  ⑤ 以图搜图（改动四）                     │
│  ┌─────────────┐  ┌─────────────┐       │   │  ├── CLIP 提取特征向量 (512维)           │
│  │  YOLOv8     │  │  Qwen       │       │   │  ├── 存入 Qdrant 向量库                   │
│  │ （本地模型） │  │ （云端 API）│       │   │  └── 向量检索 → 返回相似商品             │
│  └─────────────┘  └─────────────┘       │   └───────────────────────────────────────────┘
│        ↓ 比对结果                         │
│        ↓ 不一致时保存错误                 │
│  ┌─────────────────────────────────┐     │
│  │  data/error_data/              │     │
│  │  ├── images/（错误图片）        │     │
│  │  ├── labels/（TXT：错误/正确）  │     │
│  │  └── metadata/（每周汇总）      │     │
│  └─────────────────────────────────┘     │
└───────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│  ③ 瑕疵检测（改动二）                                                                   │
│  ├── YOLOv8 目标检测 → 识别瑕疵区域                                                     │
│  ├── 画圈标注（不同形状 + 不同颜色 = 不同程度，程度不告诉用户）                        │
│  ├── 标注图 → 仅前端展示给用户（不保存）                                               │
│  └── 瑕疵位置 + 类型 + 程度 → 结构化数据发送给 DeepSeek                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│  DeepSeek 定价辅助                                                                      │
│  ├── 接收：瑕疵位置 + 数量 + 程度等级                                                   │
│  ├── 结合：商品类别 + 市场行情                                                          │
│  └── 输出：参考价格调整建议（建议售价 + 定价理由）                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│  ④ 每周迭代训练（改动一）                                                               │
│  ├── 输入：本周错误数据 + 历史抽样数据（防过拟合）                                     │
│  ├── 操作：调用 YOLO 进行微调训练                                                      │
│  └── 输出：更新后的模型权重 (best.pt / defect_best.pt)                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```
