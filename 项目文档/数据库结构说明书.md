

# 智能二手商品发布助手 — 数据库结构说明书

## 总览

| 表名 | 行数预估 | 读写比 | 说明 |
|------|----------|--------|------|
| `users` | 小（千级） | 读多写少 | 用户账户 |
| `market_prices` | 小（百级） | 读多写极少 | 二手行情参考价 |
| `published_items` | 中（万~十万级） | 读写均衡 | **核心业务表** — 商品发布记录 |
| `ai_audit_logs` | 大（百万级） | 只写少读 | AI 调用审计日志（快速增长） |
| `hard_cases` | 小（百~千级） | 读写均衡 | YOLO 模型错题本（数据飞轮） |
| `feature_vectors` | 中（万级） | 读多写少 | CLIP 特征向量冗余备份 |
| `price_history` | 小（千级） | 读少写多 | 价格历史快照 |

## ER 关系图

```
users (1) ──────< published_items (N)     -- 一个用户可发布多个商品
  │                    │
  │                    ├──< hard_cases (N)  -- 一个商品的识别错误可被多次记录
  │                    │
  │                    └──< feature_vectors (1) -- 一个商品对应一个特征向量（1:1）
  │
  └──< ai_audit_logs (N)                   -- 一个用户的 AI 调用可有多条日志

market_prices          -- 独立表（无外键关联，仅作为行情参考数据源）
price_history          -- 独立表（无外键关联，记录历史价格变化）
```

**外键关系汇总**:

| 子表 | 外键列 | 父表 | 删除策略 |
|------|--------|------|----------|
| `published_items` | `user_id` | `users.id` | CASCADE（删用户→删商品） |
| `ai_audit_logs` | `user_id` | `users.id` | CASCADE（删用户→删日志） |
| `feature_vectors` | `item_id` | `published_items.id` | CASCADE（删商品→删向量） |

## 1. users — 用户表

存储用户账户信息。密码使用 **PBKDF2-SHA256** 加盐哈希存储。

### 表结构

| 列名 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 用户唯一 ID |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE | — | 用户名 |
| `password_hash` | VARCHAR(128) | NOT NULL | `''` | 密码哈希（格式: `salt_hex:hash_hex`） |
| `email` | VARCHAR(100) | NULL | NULL | 邮箱（预留） |
| `phone` | VARCHAR(20) | NULL | NULL | 手机号（预留） |
| `avatar_url` | VARCHAR(500) | NULL | NULL | 头像 URL（预留） |
| `status` | VARCHAR(20) | NOT NULL | `'active'` | 账户状态 |
| `role` | VARCHAR(20) | NOT NULL | `'user'` | 角色 |
| `last_login` | DATETIME | NULL | NULL | 最后登录时间 |
| `created_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP` | 注册时间 |
| `updated_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 更新时间 |

### 索引

| 索引名 | 列 | 说明 |
|--------|-----|------|
| PRIMARY | `id` | 主键 |
| `username` (UNIQUE) | `username` | 用户名唯一约束 |
| `idx_status` | `status` | 按状态查询用户 |

### 枚举值

**status**:

| 值 | 含义 |
|-----|------|
| `active` | 正常使用中 |
| `inactive` | 已停用 |
| `locked` | 已锁定（恶意行为） |
| `deleted` | 已注销（软删除） |

**role**:

| 值 | 含义 |
|-----|------|
| `user` | 普通用户 |
| `admin` | 管理员 |

### 密码存储格式

```
存储值 = hex(salt_16bytes) + ":" + hex(pbkdf2_sha256(password, salt, 100000_iterations))
示例: "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6:9f8e7d6c5b4a3928170615..."
```

> ⚠️ 不可逆哈希。忘记密码只能重置，无法找回。

## 2. market_prices — 二手行情基准表

存储各品类二手商品的市场参考价格。通过品牌+型号唯一确定一条记录。

### 表结构

| 列名 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 行情 ID |
| `category` | VARCHAR(50) | NOT NULL | — | 品类名称 |
| `brand` | VARCHAR(50) | NOT NULL | — | 品牌名称 |
| `model` | VARCHAR(100) | NOT NULL | — | 型号名称 |
| `avg_price` | DECIMAL(10,2) | NOT NULL | — | 均价（元） |
| `low_price` | DECIMAL(10,2) | NOT NULL | — | 最低参考价（元） |
| `high_price` | DECIMAL(10,2) | NOT NULL | — | 最高参考价（元） |
| `price_unit` | VARCHAR(10) | NOT NULL | `'CNY'` | 货币单位 |
| `data_source` | VARCHAR(50) | NULL | NULL | 数据来源 |
| `is_active` | TINYINT(1) | NOT NULL | `1` | 是否启用 |
| `created_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 更新时间 |

### 索引

| 索引名 | 列 | 说明 |
|--------|-----|------|
| PRIMARY | `id` | 主键 |
| `uk_brand_model` (UNIQUE) | `brand`, `model` | 品牌+型号唯一 |
| `idx_category` | `category` | 按品类筛选 |

### 当前种子数据

| 品类 | 品牌 | 型号 | 均价 |
|------|------|------|------|
| 手机 | Apple | iPhone 13 / 14 / 15 | 3200 / 4100 / 5200 |
| 手机 | HUAWEI | Mate 40 Pro / Mate 60 | 2500 / 4500 |
| 手机 | Xiaomi | 小米13 / 小米14 | 2500 / 3500 |
| 笔记本 | Apple | MacBook Air M1 / Pro 14 | 4200 / 12000 |
| 笔记本 | Lenovo | ThinkPad X1 | 3500 |
| 平板 | Apple | iPad Air 5 / Pro 12.9 | 3000 / 6000 |
| 外设 | Logitech | G610 机械键盘 | 220 |
| 耳机 | Apple / Sony | AirPods Pro 2 / WH-1000XM5 | 1000 / 2000 |
| 手表 | Apple | Watch S8 | 2500 |

### 查询策略（三层匹配）

1. **精确匹配**: `brand` + `model` 完全一致 → 直接返回
2. **模糊匹配**: 同品牌下，用 `SequenceMatcher` 比较型号相似度 → 相似度 > 70% 则返回
3. **未命中**: 返回 `None`，前端应提示用户手动输入价格

## 3. published_items — 商品发布表（核心业务表）

这是系统最核心的业务表，记录每一笔商品发布信息。贯穿整个发布流程。

### 表结构

| 列名 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 商品唯一 ID |
| `user_id` | BIGINT | NOT NULL, FK→users.id | — | 发布者 ID |
| `original_image_url` | VARCHAR(500) | NOT NULL | — | 原始图片 URL |
| `bg_removed_url` | VARCHAR(500) | NULL | NULL | 去背景图 URL |
| `annotated_url` | VARCHAR(500) | NULL | NULL | 瑕疵标注图 URL |
| `defect_count` | INT | NOT NULL | `0` | 检测到的瑕疵数量 |
| `defect_data` | JSON | NULL | NULL | 瑕疵详细数据（JSON 对象/数组） |
| `ai_generated_title` | VARCHAR(200) | NOT NULL | `''` | AI 生成的商品标题 |
| `ai_generated_desc` | TEXT | NULL | NULL | AI 生成的商品描述（长文本） |
| `suggested_price` | DECIMAL(10,2) | NULL | NULL | AI 建议售价（元） |
| `category` | VARCHAR(50) | NULL | NULL | 品类 |
| `brand` | VARCHAR(50) | NULL | NULL | 品牌 |
| `model` | VARCHAR(100) | NULL | NULL | 型号 |
| `condition` | VARCHAR(200) | NULL | NULL | 成色描述 |
| `status` | VARCHAR(20) | NOT NULL | `'draft'` | 发布状态 |
| `views` | INT | NOT NULL | `0` | 浏览次数（冗余计数） |
| `retry_count` | INT | NOT NULL | `0` | AI 生成重试次数 |
| `created_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 更新时间 |

### 索引

| 索引名 | 列 | 说明 |
|--------|-----|------|
| PRIMARY | `id` | 主键 |
| `idx_user_id` | `user_id` | 按用户查历史 |
| `idx_status` | `status` | 按状态过滤 |
| `idx_category` | `category` | 按品类过滤 |
| `idx_created_at` | `created_at` | 按时间排序 |

### 状态流转

```
draft ──→ published ──→ delisted
  ↑                       │
  └───────────────────────┘ (重新发布)
```

| 状态 | 含义 | 触发操作 |
|------|------|----------|
| `draft` | 草稿（未发布） | 用户选择"保存为草稿" |
| `published` | 已发布（商城可见） | 用户选择"发布" |
| `delisted` | 已下架（商城不可见） | 用户选择"下架" |

### `defect_data` JSON 结构

```json
[
  {
    "type": "scratch",
    "type_cn": "划痕",
    "bbox": [120, 200, 180, 260],
    "area": 3600,
    "area_ratio": 0.018,
    "confidence": 0.87,
    "severity": "moderate",
    "severity_label": "中度"
  }
]
```

### 生命周期

```
extract(上传识别) → process(瑕疵检测) → generate(生成文案) → save(保存入库)
                                                                  ↓
                                                            status=draft 或 published
                                                                  ↓
                                                      [可选] delist → publish
```

### 表结构

| 列名 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 日志 ID |
| `user_id` | BIGINT | NOT NULL, FK→users.id | — | 调用用户 ID |
| `action_type` | VARCHAR(30) | NOT NULL | — | 操作类型 |
| `model_name` | VARCHAR(50) | NOT NULL | — | 使用的模型名称 |
| `input_summary` | VARCHAR(500) | NULL | NULL | 输入摘要（截断保护隐私） |
| `raw_ai_response` | JSON | NULL | NULL | AI 原始响应（JSON 格式，截断至 2000 字符） |
| `execution_time_ms` | INT | NOT NULL | `0` | 执行耗时（毫秒） |
| `status` | VARCHAR(10) | NOT NULL | `'SUCCESS'` | 调用结果 |
| `error_message` | TEXT | NULL | NULL | 失败时的错误信息 |
| `created_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |

### 索引

| 索引名 | 列 | 说明 |
|--------|-----|------|

| `idx_action_type` | `action_type` | 按操作类型统计 |
| `idx_created_at` | `created_at` | 按时间范围查询 |

### 枚举值

**action_type**:

| 值 | 含义 | 对应接口 |
|-----|------|----------|
| `vision_extract` | 视觉识别 | `/api/v1/extract` |
| `text_generate` | 文案生成 | `/api/v1/generate` |
| `price_query` | 行情查价 | `/api/v1/price` |

> ⚠️ 此表数据增长最快（每次 AI 调用写入一条），建议定期归档或设置 TTL。

## 5. hard_cases — 错题本（数据飞轮）

记录 YOLO 本地模型与 Qwen-VL 云端模型识别结果不一致的案例，用于周期性模型重训练。

### 表结构

| 列名 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 错题 ID |
| `image_url` | VARCHAR(500) | NOT NULL | — | 错误图片 URL |
| `wrong_label` | VARCHAR(100) | NOT NULL | — | YOLO 本地模型错误分类 |
| `correct_label` | VARCHAR(100) | NOT NULL | — | Qwen-VL 正确分类（标注） |
| `model_version` | VARCHAR(50) | NULL | NULL | 当时的 YOLO 模型版本 |
| `user_id` | BIGINT | NULL | NULL | 触发用户 ID |
| `item_id` | BIGINT | NULL | NULL | 关联商品 ID |
| `confidence` | DECIMAL(5,4) | NULL | NULL | YOLO 模型输出的置信度 |
| `retry_count` | INT | NOT NULL | `0` | 同一错误重复出现的次数 |
| `is_fixed` | TINYINT(1) | NOT NULL | `0` | 是否已被修复（新模型已纠正） |
| `fixed_at` | DATETIME | NULL | NULL | 修复时间 |
| `created_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 更新时间 |

### 索引

| 索引名 | 列 | 说明 |
|--------|-----|------|
| PRIMARY | `id` | 主键 |
| `uq_error_case` (UNIQUE) | `image_url`, `wrong_label`, `correct_label` | **幂等性保证**：同一图片+错误+正确的组合只存一条 |
| `idx_wrong_label` | `wrong_label` | 按错误类型统计 |
| `idx_correct_label` | `correct_label` | 按正确类型统计 |
| `idx_is_fixed` | `is_fixed` | 筛选未修复数据 |
| `idx_created_at` | `created_at` | 按时间排序 |

### 数据飞轮工作流

```
用户上传图片
  ↓
YOLO (本地) vs Qwen-VL (云端) 结果比对
  ↓
不一致 → INSERT hard_cases（幂等，重复出现则 retry_count += 1）
  ↓
每周训练 /weekly_train → 使用 hard_cases 数据微调 YOLO 模型
  ↓
训练完成 → mark_hard_cases_fixed() → 标记 is_fixed=1
```

### 幂等性说明

唯一索引 `uq_error_case(image_url, wrong_label, correct_label)` 保证同一错误组合只存一条。重复触发时不会插入新行，而是 `retry_count += 1`。

## 6. feature_vectors — 特征向量表

### 表结构

| 列名 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | — | 特征 ID |
| `item_id` | BIGINT | NOT NULL, UNIQUE, FK→published_items.id | — | 商品 ID（1:1） |
| `feature_vector` | BLOB | NOT NULL | — | 512 维 float32 特征向量（二进制） |
| `model_name` | VARCHAR(50) | NOT NULL | `'clip_ViT-B/32'` | 提取模型名称 |
| `created_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | `CURRENT_TIMESTAMP ON UPDATE` | 更新时间 |

> ⚠️ 目前主搜索使用 Qdrant 向量数据库，此表作为备份。Qdrant 不可用时可用于恢复索引。

## 安全性说明

### 密码存储

- 使用 **PBKDF2-SHA256** 加盐哈希，100,000 次迭代
- 存储格式: `hex(salt_16bytes):hex(hash_32bytes)`
- 登录时通过 `_verify_password()` 比对哈希值
- **绝不存储明文密码**

### SQL 注入防护

- 所有 SQL 查询均使用 `%s` 参数化占位符
- `sort_by` 等动态列名使用**白名单映射**（`_SORT_COLUMNS`），禁止直接拼入 SQL
- `IN (...)` 子句的占位符数量由列表长度动态生成，不拼入用户输入

### 数据级联策略

- 删用户 → 级联删除该用户的所有商品、审计日志
- 删商品 → 级联删除该商品的特征向量
- `hard_cases` 不设外键级联，保留历史数据供模型训练

### 生产环境建议

1. **定期备份**: `published_items` 为核心资产，每日备份
2. **审计日志归档**: `ai_audit_logs` 增速最快，建议按月归档/清理
3. **密码迭代数**: 生产环境可将 PBKDF2 迭代数提升至 600,000+
4. **数据库密码**: 通过 `.env` 的 `DB_PASSWORD` 配置，不硬编码
5. **连接加密**: 生产环境建议启用 MySQL SSL/TLS

## 注意事项

1. **`defect_data` 是 JSON 列**，使用 `safe_json_dumps()` 序列化以处理 NaN/Infinity 等特殊值。反序列化时注意异常处理。

2. **`hard_cases` 的幂等性**依赖唯一索引 `uq_error_case`，插入时由 CRUD 层手动实现 ON DUPLICATE 逻辑（先查后插/更）。

3. **`feature_vectors` 与 Qdrant 的关系**：Qdrant 是主搜索引擎，此表为冷备份。正常情况下搜索走 Qdrant，此表仅在重建索引时使用。

4. **DECIMAL(10,2)** 表示最多 10 位数字，其中 2 位小数（即最大 99,999,999.99 元）。对二手商品足够，但若扩展到汽车/房产等大额商品需调整精度。

5. **`condition` 作为列名**在 MySQL 中是保留字，SQL 中必须用反引号包裹：`` `condition` ``。

6. **`last_login` 字段**在用户登录时由 `authenticate_user()` 更新，不设触发器。
