## 📄 表结构详细说明


### 1. users —— 用户信息表

存储所有注册用户的基本信息。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 用户唯一标识 |
| `username` | VARCHAR(50) | NOT NULL UNIQUE | 用户名 |
| `password_hash` | VARCHAR(128) | NOT NULL | PBKDF2 密码哈希 |
| `email` | VARCHAR(100) | NULL | 电子邮箱 |
| `phone` | VARCHAR(20) | NULL | 手机号 |
| `avatar_url` | VARCHAR(500) | NULL | 头像URL |
| `status` | VARCHAR(20) | NOT NULL DEFAULT 'active' | 状态: active/inactive/locked/deleted |
| `role` | VARCHAR(20) | NOT NULL DEFAULT 'user' | 角色: user/admin |
| `last_login` | DATETIME | NULL | 最后登录时间 |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| `updated_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_username | username | 唯一索引 |
| idx_status | status | 普通索引 |

**说明：**
- 密码使用 PBKDF2-SHA256 算法，100,000 次迭代，16 字节随机盐
- 存储格式：`salt_hex:hash_hex`
- `status` 用于账号禁用/锁定/删除等管理操作


### 2. market_prices —— 二手行情基准表

存储二手商品的市场参考价格，用于价格查询和定价参考。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 行情唯一标识 |
| `category` | VARCHAR(50) | NOT NULL | 品类（手机/笔记本/平板等） |
| `brand` | VARCHAR(50) | NOT NULL | 品牌 |
| `model` | VARCHAR(100) | NOT NULL | 型号 |
| `avg_price` | DECIMAL(10,2) | NOT NULL | 二手市场均价 |
| `low_price` | DECIMAL(10,2) | NOT NULL | 最低参考价 |
| `high_price` | DECIMAL(10,2) | NOT NULL | 最高参考价 |
| `price_unit` | VARCHAR(10) | NOT NULL DEFAULT 'CNY' | 货币单位 |
| `data_source` | VARCHAR(50) | NULL | 数据来源 |
| `is_active` | TINYINT(1) | NOT NULL DEFAULT 1 | 是否启用 |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_brand_model | brand, model | 唯一索引 |
| idx_category | category | 普通索引 |

**说明：**
- `(brand, model)` 唯一索引确保同品牌同型号只有一条记录
- 支持三层匹配策略：精确匹配 → 模糊匹配（difflib > 0.7）→ 返回空


### 3. published_items —— 商品发布表（核心业务表）

存储用户发布的商品信息，是系统最核心的业务表。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 商品唯一标识 |
| `user_id` | BIGINT | NOT NULL | 发布者ID（关联 users.id） |
| `original_image_url` | VARCHAR(500) | NOT NULL | 原始图片URL |
| `bg_removed_url` | VARCHAR(500) | NULL | 去背景图URL |
| `annotated_url` | VARCHAR(500) | NULL | 瑕疵标注图URL |
| `defect_count` | INT | NOT NULL DEFAULT 0 | 瑕疵数量 |
| `defect_data` | JSON | NULL | 瑕疵详细数据（位置/程度） |
| `ai_generated_title` | VARCHAR(200) | NOT NULL DEFAULT '' | AI生成标题 |
| `ai_generated_desc` | TEXT | NULL | AI生成描述 |
| `suggested_price` | DECIMAL(10,2) | NULL | 建议价格 |
| `category` | VARCHAR(50) | NULL | 品类（手机/笔记本/平板等） |
| `brand` | VARCHAR(50) | NULL | 品牌（Apple/华为/小米等） |
| `model` | VARCHAR(100) | NULL | 型号（iPhone 13/Mate 60等） |
| `condition` | VARCHAR(200) | NULL | 成色描述 |
| `status` | VARCHAR(20) | NOT NULL DEFAULT 'draft' | 状态: draft/published/hidden/sold/deleted |
| `views` | INT | NOT NULL DEFAULT 0 | 浏览次数 |
| `likes` | INT | NOT NULL DEFAULT 0 | 点赞数 |
| `retry_count` | INT | NOT NULL DEFAULT 0 | 重试次数 |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_user_id | user_id | 普通索引 |
| idx_status | status | 普通索引 |
| idx_category | category | 普通索引 |
| idx_created_at | created_at | 普通索引 |

**外键：**

| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_published_items_user | user_id | users.id | ON DELETE CASCADE |

**状态流转：**

```
draft (草稿)
    │
    ▼
published (已发布) ←→ delisted (已下架)
    │
    ▼
sold (已售出) / deleted (已删除)
```

**说明：**
- `defect_data` 存储 JSON 数组，每个元素包含瑕疵类型、位置、程度等级
- `condition` 字段统一为成色描述（区别于 `item_condition`，已统一）
- `retry_count` 用于异步任务重试计数


### 4. ai_audit_logs —— AI 审计日志表

记录所有 AI 模型调用的完整链路信息，用于问题追溯和成本核算。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 日志唯一标识 |
| `user_id` | BIGINT | NOT NULL | 用户ID（关联 users.id） |
| `action_type` | VARCHAR(30) | NOT NULL | 操作类型: vision_extract/text_generate/price_query/defect_detect |
| `model_name` | VARCHAR(50) | NOT NULL | 模型名称 |
| `input_summary` | VARCHAR(500) | NULL | 输入内容摘要 |
| `raw_ai_response` | JSON | NULL | AI原始响应（JSON格式） |
| `execution_time_ms` | INT | NOT NULL DEFAULT 0 | 执行耗时（毫秒） |
| `status` | VARCHAR(10) | NOT NULL DEFAULT 'SUCCESS' | 状态: SUCCESS/FAILED |
| `error_message` | TEXT | NULL | 错误信息 |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_user_id | user_id | 普通索引 |
| idx_status | status | 普通索引 |
| idx_action_type | action_type | 普通索引 |
| idx_created_at | created_at | 普通索引 |

**外键：**

| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_audit_logs_user | user_id | users.id | ON DELETE CASCADE |

**action_type 枚举：**

| 值 | 说明 |
|:---|:---|
| `vision_extract` | Qwen 视觉识别 |
| `text_generate` | DeepSeek 文案生成 |
| `price_query` | 市场行情查询 |
| `defect_detect` | YOLO 瑕疵检测 |

**说明：**
- `raw_ai_response` 使用 `safe_json_dumps()` 序列化，避免双编码
- 记录执行耗时用于性能监控和成本分析


### 5. item_likes —— 点赞记录表

存储用户对商品的点赞记录，支持取消点赞。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 点赞唯一标识 |
| `item_id` | BIGINT | NOT NULL | 商品ID（关联 published_items.id） |
| `user_id` | BIGINT | NOT NULL | 用户ID（关联 users.id） |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 点赞时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_item_user | item_id, user_id | 唯一索引 |
| idx_item_id | item_id | 普通索引 |
| idx_user_id | user_id | 普通索引 |

**外键：**

| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_likes_item | item_id | published_items.id | ON DELETE CASCADE |
| fk_likes_user | user_id | users.id | ON DELETE CASCADE |

**说明：**
- `(item_id, user_id)` 唯一索引防止重复点赞
- 点赞/取消点赞使用显式事务保证 `item_likes` 和 `published_items.likes` 计数一致


### 6. hard_cases —— 错题本（错误数据收集）

存储双模型比对不一致的错误数据，用于数据飞轮和模型迭代训练。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 错题唯一标识 |
| `image_url` | VARCHAR(500) | NOT NULL | 错误图片URL |
| `wrong_label` | VARCHAR(100) | NOT NULL | 本地模型错误分类 |
| `correct_label` | VARCHAR(100) | NOT NULL | Qwen 正确分类 |
| `model_version` | VARCHAR(50) | NULL | 当时模型版本 |
| `user_id` | BIGINT | NULL | 触发用户ID |
| `item_id` | BIGINT | NULL | 关联商品ID |
| `confidence` | DECIMAL(5,4) | NULL | 模型置信度 |
| `retry_count` | INT | NOT NULL DEFAULT 0 | 重试次数 |
| `is_fixed` | TINYINT(1) | NOT NULL DEFAULT 0 | 是否已修复 |
| `fixed_at` | DATETIME | NULL | 修复时间 |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uq_error_case | image_url, wrong_label, correct_label | 唯一索引 |
| idx_wrong_label | wrong_label | 普通索引 |
| idx_correct_label | correct_label | 普通索引 |
| idx_is_fixed | is_fixed | 普通索引 |
| idx_created_at | created_at | 普通索引 |

**说明：**
- `uq_error_case` 唯一索引防止重复插入相同错误
- `confidence` 记录模型置信度，用于数据筛选
- `retry_count` 记录重试次数
- `is_fixed` 标记是否已被训练使用，形成数据飞轮闭环


### 7. feature_vectors —— 特征向量表（以图搜图备用）

存储 CLIP 提取的图片特征向量，用于以图搜图（Qdrant 为主，此表为备用）。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 特征唯一标识 |
| `item_id` | BIGINT | NOT NULL UNIQUE | 商品ID（关联 published_items.id） |
| `feature_vector` | BLOB | NOT NULL | 特征向量（二进制） |
| `model_name` | VARCHAR(50) | NOT NULL DEFAULT 'clip_ViT-B/32' | 特征提取模型 |
| `created_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_item_id | item_id | 唯一索引 |

**外键：**

| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_feature_item | item_id | published_items.id | ON DELETE CASCADE |

**说明：**
- 主用以图搜图使用 Qdrant 向量数据库，此表为 FAISS 备用方案
- `feature_vector` 存储 512 维 CLIP 特征向量的二进制序列化


### 8. price_history —— 价格历史表

存储商品价格的历史变化，用于价格趋势分析。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 历史唯一标识 |
| `brand` | VARCHAR(50) | NOT NULL | 品牌 |
| `model` | VARCHAR(100) | NOT NULL | 型号 |
| `price` | DECIMAL(10,2) | NOT NULL | 价格 |
| `price_type` | VARCHAR(20) | NOT NULL DEFAULT 'avg' | 类型: avg/low/high |
| `source` | VARCHAR(50) | NULL | 数据来源 |
| `recorded_at` | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 记录时间 |

**索引：**

| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_brand_model | brand, model | 普通索引 |
| idx_recorded_at | recorded_at | 普通索引 |

**说明：**
- 用于追踪价格变化趋势，后续可做价格预测
- `price_type` 区分均价/最低价/最高价


## 🔗 表关系汇总

| 主表 | 关联表 | 关系 | 外键字段 | 说明 |
|:---|:---|:---:|:---|:---|
| users | published_items | 1:N | user_id | 一个用户发布多个商品 |
| users | ai_audit_logs | 1:N | user_id | 一个用户有多条审计日志 |
| users | item_likes | 1:N | user_id | 一个用户点赞多个商品 |
| users | hard_cases | 1:N | user_id | 一个用户触发多个错题 |
| published_items | item_likes | 1:N | item_id | 一个商品被多人点赞 |
| published_items | feature_vectors | 1:1 | item_id | 一个商品对应一个特征向量 |
| published_items | hard_cases | 1:N | item_id | 一个商品可能产生多个错题 |
| published_items | users | N:1 | user_id | 多个商品属于一个用户 |
