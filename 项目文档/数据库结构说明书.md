

## 📄 表结构详细说明


### 1. users —— 用户信息表

存储所有注册用户的基本信息。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 用户唯一标识 |
| `username` | VARCHAR(50) | NOT NULL UNIQUE | 用户名 |
| `password_hash` | VARCHAR(128) | NOT NULL | PBKDF2 密码哈希 |
| `email` | VARCHAR(100) | NULL | 电子邮箱 |
| `phone` | VARCHAR(20) | NULL | 手机号 |
| `avatar_url` | VARCHAR(500) | NULL | 头像URL |
| `status` | VARCHAR(20) | DEFAULT 'active' | active/inactive/locked/deleted |
| `role` | VARCHAR(20) | DEFAULT 'user' | user/admin |
| `last_login` | DATETIME | NULL | 最后登录时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_username | username | 唯一索引 |
| idx_status | status | 普通索引 |


### 2. market_prices —— 二手行情基准表

存储二手商品的市场参考价格，用于价格查询和定价参考。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 行情唯一标识 |
| `category` | VARCHAR(50) | NOT NULL | 品类（手机/笔记本/平板等） |
| `brand` | VARCHAR(50) | NOT NULL | 品牌 |
| `model` | VARCHAR(100) | NOT NULL | 型号 |
| `avg_price` | DECIMAL(10,2) | NOT NULL | 二手市场均价 |
| `low_price` | DECIMAL(10,2) | NOT NULL | 最低参考价 |
| `high_price` | DECIMAL(10,2) | NOT NULL | 最高参考价 |
| `price_unit` | VARCHAR(10) | DEFAULT 'CNY' | 货币单位 |
| `data_source` | VARCHAR(50) | NULL | 数据来源 |
| `is_active` | TINYINT(1) | DEFAULT 1 | 是否启用 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_brand_model | brand, model | 唯一索引 |
| idx_category | category | 普通索引 |


### 3. published_items —— 商品发布表

核心业务表，存储用户发布的商品信息。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 商品唯一标识 |
| `user_id` | BIGINT | NOT NULL | 发布者ID（关联users.id） |
| `original_image_url` | VARCHAR(500) | NOT NULL | 原始图片URL |
| `bg_removed_url` | VARCHAR(500) | NULL | 去背景图URL |
| `annotated_url` | VARCHAR(500) | NULL | 瑕疵标注图URL |
| `defect_count` | INT | DEFAULT 0 | 瑕疵数量 |
| `defect_types` | JSON | NULL | 瑕疵类型列表 |
| `defect_data` | JSON | NULL | 瑕疵详细数据 |
| `ai_generated_title` | VARCHAR(200) | DEFAULT '' | AI生成标题 |
| `ai_generated_desc` | TEXT | NULL | AI生成描述 |
| `suggested_price` | DECIMAL(10,2) | NULL | 建议价格 |
| `category` | VARCHAR(50) | NULL | 品类 |
| `item_condition` | VARCHAR(20) | DEFAULT 'unknown' | 成色 |
| `status` | VARCHAR(20) | DEFAULT 'draft' | draft/published/hidden/sold/deleted |
| `views` | INT | DEFAULT 0 | 浏览次数 |
| `likes` | INT | DEFAULT 0 | 点赞数 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_user_id | user_id | 普通索引 |
| idx_status | status | 普通索引 |
| idx_category | category | 普通索引 |

**外键：**
| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_published_items_user | user_id | users.id | ON DELETE CASCADE |


### 4. ai_audit_logs —— AI审计日志表

记录所有AI模型调用的完整链路信息。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 日志唯一标识 |
| `user_id` | BIGINT | NOT NULL | 用户ID（关联users.id） |
| `action_type` | VARCHAR(30) | NOT NULL | vision_extract/text_generate/price_query |
| `model_name` | VARCHAR(50) | NOT NULL | 模型名称 |
| `input_summary` | VARCHAR(500) | NULL | 输入内容摘要 |
| `raw_ai_response` | JSON | NULL | AI原始响应 |
| `execution_time_ms` | INT | DEFAULT 0 | 执行耗时（毫秒） |
| `status` | VARCHAR(10) | DEFAULT 'SUCCESS' | SUCCESS/FAILED |
| `error_message` | TEXT | NULL | 错误信息 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_user_id | user_id | 普通索引 |
| idx_status | status | 普通索引 |
| idx_action_type | action_type | 普通索引 |
| idx_created_at | created_at | 普通索引 |

**外键：**
| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_audit_logs_user | user_id | users.id | ON DELETE CASCADE |


### 5. item_likes —— 点赞记录表

存储用户对商品的点赞记录。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 点赞唯一标识 |
| `item_id` | BIGINT | NOT NULL | 商品ID（关联published_items.id） |
| `user_id` | BIGINT | NOT NULL | 用户ID（关联users.id） |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 点赞时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_item_user | item_id, user_id | 唯一索引 |
| idx_item_id | item_id | 普通索引 |
| idx_user_id | user_id | 普通索引 |

**外键：**
| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_likes_item | item_id | published_items.id | ON DELETE CASCADE |
| fk_likes_user | user_id | users.id | ON DELETE CASCADE |


### 6. hard_cases —— 错题本

存储双模型比对不一致的错误数据，用于模型迭代训练。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 错题唯一标识 |
| `image_url` | VARCHAR(500) | NOT NULL | 错误图片URL |
| `wrong_label` | VARCHAR(100) | NOT NULL | 本地模型错误分类 |
| `correct_label` | VARCHAR(100) | NOT NULL | Qwen正确分类 |
| `model_version` | VARCHAR(50) | NULL | 当时模型版本 |
| `user_id` | BIGINT | NULL | 触发用户ID |
| `item_id` | BIGINT | NULL | 关联商品ID |
| `is_fixed` | TINYINT(1) | DEFAULT 0 | 是否已修复 |
| `fixed_at` | DATETIME | NULL | 修复时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_wrong_label | wrong_label | 普通索引 |
| idx_correct_label | correct_label | 普通索引 |
| idx_is_fixed | is_fixed | 普通索引 |


### 7. feature_vectors —— 特征向量表

存储以图搜图所需的图片特征向量（备用于FAISS/Qdrant）。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 特征唯一标识 |
| `item_id` | BIGINT | NOT NULL UNIQUE | 商品ID（关联published_items.id） |
| `feature_vector` | BLOB | NOT NULL | 特征向量（二进制） |
| `model_name` | VARCHAR(50) | DEFAULT 'mobilenet_v3' | 特征提取模型 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| uk_item_id | item_id | 唯一索引 |

**外键：**
| 约束名 | 字段 | 关联表 | 操作 |
|:---|:---|:---|:---|
| fk_feature_item | item_id | published_items.id | ON DELETE CASCADE |


### 8. price_history —— 价格历史表

存储商品价格的历史变化，用于价格趋势分析。

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| `id` | BIGINT | PRIMARY KEY AUTO_INCREMENT | 历史唯一标识 |
| `brand` | VARCHAR(50) | NOT NULL | 品牌 |
| `model` | VARCHAR(100) | NOT NULL | 型号 |
| `price` | DECIMAL(10,2) | NOT NULL | 价格 |
| `price_type` | VARCHAR(20) | DEFAULT 'avg' | avg/low/high |
| `source` | VARCHAR(50) | NULL | 数据来源 |
| `recorded_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 记录时间 |

**索引：**
| 索引名 | 字段 | 类型 |
|:---|:---|:---|
| PRIMARY | id | 主键 |
| idx_brand_model | brand, model | 普通索引 |
| idx_recorded_at | recorded_at | 普通索引 |

|:---|:---|:---:|:---|

## 🔗 建表SQL

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS `market_transactions`
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE `market_transactions`;

-- 1. users
DROP TABLE IF EXISTS `users`;
CREATE TABLE `users` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
  `password_hash` VARCHAR(128) NOT NULL DEFAULT '' COMMENT '密码哈希',
  `email` VARCHAR(100) NULL DEFAULT NULL COMMENT '邮箱',
  `phone` VARCHAR(20) NULL DEFAULT NULL COMMENT '手机号',
  `avatar_url` VARCHAR(500) NULL DEFAULT NULL COMMENT '头像URL',
  `status` VARCHAR(20) NOT NULL DEFAULT 'active' COMMENT 'active/inactive/locked/deleted',
  `role` VARCHAR(20) NOT NULL DEFAULT 'user' COMMENT 'user/admin',
  `last_login` DATETIME NULL DEFAULT NULL COMMENT '最后登录时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';

-- 2. market_prices
DROP TABLE IF EXISTS `market_prices`;
CREATE TABLE `market_prices` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '行情ID',
  `category` VARCHAR(50) NOT NULL COMMENT '品类',
  `brand` VARCHAR(50) NOT NULL COMMENT '品牌',
  `model` VARCHAR(100) NOT NULL COMMENT '型号',
  `avg_price` DECIMAL(10,2) NOT NULL COMMENT '均价',
  `low_price` DECIMAL(10,2) NOT NULL COMMENT '最低参考价',
  `high_price` DECIMAL(10,2) NOT NULL COMMENT '最高参考价',
  `price_unit` VARCHAR(10) NOT NULL DEFAULT 'CNY' COMMENT '货币单位',
  `data_source` VARCHAR(50) NULL DEFAULT NULL COMMENT '数据来源',
  `is_active` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_brand_model` (`brand`, `model`),
  INDEX `idx_category` (`category`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='二手行情表';

-- 3. published_items
DROP TABLE IF EXISTS `published_items`;
CREATE TABLE `published_items` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '商品ID',
  `user_id` BIGINT NOT NULL COMMENT '发布者ID',
  `original_image_url` VARCHAR(500) NOT NULL COMMENT '原始图片URL',
  `bg_removed_url` VARCHAR(500) NULL DEFAULT NULL COMMENT '去背景图URL',
  `annotated_url` VARCHAR(500) NULL DEFAULT NULL COMMENT '标注图URL',
  `defect_count` INT NOT NULL DEFAULT 0 COMMENT '瑕疵数量',
  `defect_types` JSON NULL DEFAULT NULL COMMENT '瑕疵类型列表',
  `defect_data` JSON NULL DEFAULT NULL COMMENT '瑕疵详细数据',
  `ai_generated_title` VARCHAR(200) NOT NULL DEFAULT '' COMMENT 'AI生成标题',
  `ai_generated_desc` TEXT NULL COMMENT 'AI生成描述',
  `suggested_price` DECIMAL(10,2) NULL DEFAULT NULL COMMENT '建议价格',
  `category` VARCHAR(50) NULL DEFAULT NULL COMMENT '品类',
  `item_condition` VARCHAR(20) NOT NULL DEFAULT 'unknown' COMMENT '成色',
  `status` VARCHAR(20) NOT NULL DEFAULT 'draft' COMMENT 'draft/published/hidden/sold/deleted',
  `views` INT NOT NULL DEFAULT 0 COMMENT '浏览次数',
  `likes` INT NOT NULL DEFAULT 0 COMMENT '点赞数',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_category` (`category`),
  CONSTRAINT `fk_published_items_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='商品发布表';

-- 4. ai_audit_logs
DROP TABLE IF EXISTS `ai_audit_logs`;
CREATE TABLE `ai_audit_logs` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '日志ID',
  `user_id` BIGINT NOT NULL COMMENT '用户ID',
  `action_type` VARCHAR(30) NOT NULL COMMENT 'vision_extract/text_generate/price_query',
  `model_name` VARCHAR(50) NOT NULL COMMENT '模型名称',
  `input_summary` VARCHAR(500) NULL COMMENT '输入摘要',
  `raw_ai_response` JSON NULL COMMENT 'AI原始响应',
  `execution_time_ms` INT NOT NULL DEFAULT 0 COMMENT '耗时(毫秒)',
  `status` VARCHAR(10) NOT NULL DEFAULT 'SUCCESS' COMMENT 'SUCCESS/FAILED',
  `error_message` TEXT NULL COMMENT '错误信息',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_action_type` (`action_type`),
  INDEX `idx_created_at` (`created_at`),
  CONSTRAINT `fk_audit_logs_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='AI审计日志表';

-- 5. item_likes
DROP TABLE IF EXISTS `item_likes`;
CREATE TABLE `item_likes` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '点赞ID',
  `item_id` BIGINT NOT NULL COMMENT '商品ID',
  `user_id` BIGINT NOT NULL COMMENT '用户ID',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '点赞时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_item_user` (`item_id`, `user_id`),
  INDEX `idx_item_id` (`item_id`),
  INDEX `idx_user_id` (`user_id`),
  CONSTRAINT `fk_likes_item` FOREIGN KEY (`item_id`) REFERENCES `published_items` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_likes_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='点赞表';

-- 6. hard_cases
DROP TABLE IF EXISTS `hard_cases`;
CREATE TABLE `hard_cases` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '错题ID',
  `image_url` VARCHAR(500) NOT NULL COMMENT '错误图片URL',
  `wrong_label` VARCHAR(100) NOT NULL COMMENT '错误分类',
  `correct_label` VARCHAR(100) NOT NULL COMMENT '正确分类',
  `model_version` VARCHAR(50) NULL DEFAULT NULL COMMENT '模型版本',
  `user_id` BIGINT NULL DEFAULT NULL COMMENT '用户ID',
  `item_id` BIGINT NULL DEFAULT NULL COMMENT '商品ID',
  `is_fixed` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否已修复',
  `fixed_at` DATETIME NULL DEFAULT NULL COMMENT '修复时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  INDEX `idx_wrong_label` (`wrong_label`),
  INDEX `idx_correct_label` (`correct_label`),
  INDEX `idx_is_fixed` (`is_fixed`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='错题本';

-- 7. feature_vectors
DROP TABLE IF EXISTS `feature_vectors`;
CREATE TABLE `feature_vectors` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '特征ID',
  `item_id` BIGINT NOT NULL UNIQUE COMMENT '商品ID',
  `feature_vector` BLOB NOT NULL COMMENT '特征向量(二进制)',
  `model_name` VARCHAR(50) NOT NULL DEFAULT 'mobilenet_v3' COMMENT '特征提取模型',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  CONSTRAINT `fk_feature_item` FOREIGN KEY (`item_id`) REFERENCES `published_items` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='特征向量表';

-- 8. price_history
DROP TABLE IF EXISTS `price_history`;
CREATE TABLE `price_history` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '历史ID',
  `brand` VARCHAR(50) NOT NULL COMMENT '品牌',
  `model` VARCHAR(100) NOT NULL COMMENT '型号',
  `price` DECIMAL(10,2) NOT NULL COMMENT '价格',
  `price_type` VARCHAR(20) NOT NULL DEFAULT 'avg' COMMENT 'avg/low/high',
  `source` VARCHAR(50) NULL DEFAULT NULL COMMENT '数据来源',
  `recorded_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录时间',
  PRIMARY KEY (`id`),
  INDEX `idx_brand_model` (`brand`, `model`),
  INDEX `idx_recorded_at` (`recorded_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='价格历史表';
