# 智能二手商品发布助手 — 前端接口文档

## 📋 基本信息

| 项目 | 说明 |
|:---|:---|
| **Base URL** | `http://localhost:8000/api/v1` |
| **Content-Type** | `application/json`（文件上传用 `multipart/form-data`） |
| **响应格式** | JSON |
| **SSE 流式** | `text/event-stream`，格式 `data: <JSON>\n\n` |


## 🔐 认证说明

登录后获得 JWT Token，存入 `localStorage`。需要认证的请求携带 Header：

```
Authorization: Bearer <token>
```

| 配置项 | 说明 |
|:---|:---|
| **Token 有效期** | 30 分钟 |
| **存储方式** | `localStorage.setItem('token', data.access_token)` |
| **刷新策略** | 过期后需重新登录 |


## 📤 文件上传规范

| 项目 | 限制 |
|:---|:---|
| **支持格式** | JPG / PNG / WEBP |
| **单文件大小** | ≤ 10MB |
| **请求方式** | `multipart/form-data` |


## ❌ 错误响应格式

**非 2xx 响应：**

```json
{ "detail": "错误描述" }
```

**HTTP 状态码说明：**

| 状态码 | 含义 |
|:---:|:---|
| 400 | 参数错误 |
| 401 | 未认证 / Token 过期 |
| 403 | 无权操作（如操作他人商品） |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用 |


## 1. 👤 用户认证

### POST `/api/v1/register` — 注册

> **认证要求：** ❌ 无需认证

**请求体：**

```json
{
  "username": "testuser",
  "password": "test123456"
}
```

**参数校验：**

| 字段 | 校验规则 |
|:---|:---|
| username | ≥ 2 个字符 |
| password | ≥ 6 位 |

**响应（成功）：**

```json
{
  "success": true,
  "user_id": 1,
  "username": "testuser",
  "message": "注册成功"
}
```

> ⚠️ 注册不返回 token，需再调用登录接口。


### POST `/api/v1/login` — 登录

> **认证要求：** ❌ 无需认证

**请求体：**

```json
{
  "username": "testuser",
  "password": "test123456"
}
```

**响应（成功）：**

```json
{
  "success": true,
```

## 3. ✍️ 文案生成（SSE 流式）

### POST `/api/v1/generate` — 流式生成文案

> **认证要求：** 🔒 需要认证

**请求体：**

```json
{
  "category": "手机",
  "brand": "Apple",
  "model": "iPhone 14 Pro",
  "condition": "9成新，屏幕无划痕",
  "image_urls": [],
  "avg_price": 6200,
  "low_price": 5500,
  "high_price": 7000
}
```

**SSE 事件流（4 种事件类型）：**

**① 开始事件**

```
data: {"type":"start","status":"started","message":"开始生成文案..."}
```

**② 内容事件**

```
data: {"type":"content","content":"这是一台"}
data: {"type":"content","content":"9成新的"}
data: {"type":"content","content":"iPhone 14 Pro"}
```

**③ 完成事件**

```
data: {"type":"done","status":"success","message":"文案生成完成"}
```

**④ 错误事件**

```
data: {"type":"error","status":"error","code":"GENERATION_ERROR","message":"生成失败，请重试"}
```

**前端处理示例：**

```javascript
const eventSource = new EventSource('/api/v1/generate')
// 或使用 fetch + ReadableStream
```

## 🚀 典型前端开发流程

### 完整发布流程

```
用户上传图片
    ↓
POST /extract
    ↓
获得 { category, brand, model, condition }
    ↓
GET /price?brand=Apple&model=iPhone
    ↓
获得 { avg_price, low_price, high_price }
    ↓
用户确认/编辑信息
    ↓
POST /generate (SSE 流式)
    ↓
展示打字机效果文案
    ↓
用户确认
    ↓
POST /history/save
    ↓
发布完成 ✅
```


### 瑕疵检测流程

```
用户上传图片
    ↓
POST /process/image
    ↓
获得：
  • 瑕疵标注图 (annotated_base64)
  • 瑕疵列表 (defects)
  • AI 定价建议 (price_suggestion)
    ↓
前端展示标注图 + 定价建议
    ↓
用户确认发布
```


### 以图搜图流程

```
用户上传图片/输入文字
    ↓
POST /search/image 或 /search/text
    ↓
获得相似商品列表 + 相似度分数
    ↓
展示匹配结果
    ↓
点击商品 → 跳转详情页
```
