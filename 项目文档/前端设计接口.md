

# 智能二手商品发布助手 — 前端接口文档

## 通用规范

### 认证机制

| 项目 | 说明 |
|------|------|
| 认证方式 | HTTP Bearer Token（JWT） |
| Token 获取 | 登录接口返回 `access_token` |
| Token 存放 | 前端存入 `localStorage`，请求时带 `Authorization: Bearer <token>` |
| Token 有效期 | 默认 30 分钟（可在 `.env` 中配置 `ACCESS_TOKEN_EXPIRE_MINUTES`） |
| 强制认证接口 | 除 `/market`、`/price`、`/health`、`/` 外，其他接口均需登录 |
| 401 处理 | 前端拦截器检测 401 → 清除 token → 跳转登录页 |

### 速率限制

每个端点有独立的速率限制（按 IP + 端点），超出限制返回 `429 Too Many Requests`。

| 端点 | 限制 | 说明 |
|------|------|------|
| `/api/v1/register` | 5 次/分钟 | 防恶意注册 |
| `/api/v1/login` | 10 次/分钟 | 防暴力破解 |
| `/api/v1/extract` | 10 次/分钟 | Qwen-VL 付费调用 |
| `/api/v1/generate` | 20 次/分钟 | DeepSeek 付费调用 |
| `/api/v1/process/image` | 30 次/分钟 | 计算密集型 |
| `/api/v1/yolo/detect` | 60 次/分钟 | 本地模型 |
| `/api/v1/search/image` | 60 次/分钟 | 向量搜索 |
| `/api/v1/search/text` | 120 次/分钟 | 向量搜索 |

**429 响应格式**：
```json
{
  "detail": {
    "error": "请求过于频繁，请稍后再试",
    "retry_after_seconds": 5.2,
    "limit": "10次/60秒"
  }
}
```
响应头中带有 `Retry-After`（秒），前端应据此做退避等待。

### 统一错误格式

```json
// 400 参数错误
{ "detail": "用户名至少 2 个字符" }

// 401 未认证
{ "detail": "无效的认证凭证" }

// 403 无权限
{ "detail": "无权操作此商品" }

// 404
{ "detail": "商品不存在" }

// 429 频率限制
{ "detail": { "error": "请求过于频繁，请稍后再试", "retry_after_seconds": 5.2 } }

// 500 服务端错误
{ "detail": "处理失败: xxx" }
```

## 1. 用户认证

### 1.1 注册

```
POST /api/v1/register
```

**认证**: 无需  
**速率限制**: 5 次/分钟

**请求体** (JSON):
```json
{
  "username": "testuser",
  "password": "mypassword"
}
```

| 字段 | 类型 | 必填 | 校验 |
|------|------|------|------|
| username | string | 是 | ≥ 2 字符 |
| password | string | 是 | ≥ 6 位 |

**成功响应** (200):
```json
{
  "success": true,
  "user_id": 3,
  "username": "testuser",
  "message": "注册成功"
}
```

**失败响应** (400):
```json
{ "detail": "用户名已被注册" }
// 或
{ "detail": "用户名至少 2 个字符" }
// 或
{ "detail": "密码至少 6 位" }
```

### 1.2 登录

```
POST /api/v1/login
```

**认证**: 无需  
**速率限制**: 10 次/分钟

**请求体** (JSON):
```json
{
  "username": "testuser",
  "password": "mypassword"
}
```

**成功响应** (200):
```json
{
  "success": true,
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "user_id": 3,
  "username": "testuser",
```

## 2. 视觉识别

上传一张商品图片，调用 Qwen-VL-Max 大模型识别商品信息。

```
POST /api/v1/extract
```

**认证**: 需要（JWT Bearer）  
**速率限制**: 10 次/分钟  
**Content-Type**: `multipart/form-data`

**请求参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| image | File | 是 | 商品图片文件（≤10MB，支持 JPG/PNG/WebP） |

**成功响应** (200):
```json
{
  "success": true,
  "data": {
    "category": "手机",
    "brand": "Apple",
    "model": "iPhone 15 Pro",
    "condition": "外观95新，屏幕无划痕，边框轻微使用痕迹"
  },
  "image_urls": [
    "/static/uploads/a1b2c3d4e5f6.jpg"
  ],
  "error": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| success | bool | 是否识别成功 |
| data.category | string | 品类（手机/笔记本/平板/耳机/手表/外设 等） |
| data.brand | string | 品牌（Apple/华为/小米/Sony 等） |
| data.model | string | 型号 |
| data.condition | string | 成色描述 |
| image_urls | string[] | 上传后的图片 URL 列表（数组长度 1） |
| error | string\|null | 识别失败时的错误信息（此时 data 为空对象） |

**失败时仍有响应**（success=false）:
```json
{
  "success": false,
  "data": { "category": "", "brand": "", "model": "", "condition": "" },
  "image_urls": ["/static/uploads/xxx.jpg"],
  "error": "API 调用超时"
}
```

> ⚠️ **注意**: 即使 Qwen-VL 识别失败，图片也已保存到服务器。`image_urls[0]` 仍然有效，可用于后续流程。

## 3. 行情查价

根据品牌+型号查询二手市场价格行情。

```
GET /api/v1/price
```

**认证**: 可选（未登录也可查询，但推荐带 token）  
**速率限制**: 无独立限制

**查询参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| brand | string | 是 | 品牌名，如 `Apple`、`HUAWEI` |
| model | string | 是 | 型号名，如 `iPhone 15`、`Mate 60` |

**示例请求**:
```
GET /api/v1/price?brand=Apple&model=iPhone%2015
```

**成功响应** (200):
```json
{
  "category": "手机",
  "brand": "Apple",
  "model": "iPhone 15",
  "avg_price": 5200.00,
  "low_price": 4800.00,
  "high_price": 5800.00,
  "matched": true,
  "note": "精确匹配"
}
```

`matched` 的三种情况：

| matched | note | 说明 |
|---------|------|------|
| true | "精确匹配" | 品牌+型号完全匹配到数据库记录 |
| true | "模糊匹配: xxx" | 用相似度算法匹配到最接近的型号 |
| false | "未找到行情数据" | 数据库中无此品牌/型号 |

> ⚠️ 当 `matched=false` 时，`avg_price/low_price/high_price` 均为 `0`，前端应提示用户手动输入价格。

## 4. 文案生成（SSE 流式）

根据商品信息，调用 DeepSeek 大模型流式生成带货文案。

```
POST /api/v1/generate
```

**认证**: 需要（JWT Bearer）  
**速率限制**: 20 次/分钟  
**响应格式**: `text/event-stream` (SSE)

### 请求

**请求体** (JSON):
```json
{
  "user_id": 3,
  "category": "手机",
  "brand": "Apple",
  "model": "iPhone 15 Pro",
  "condition": "95新，屏幕无划痕",
  "image_urls": ["/static/uploads/xxx.jpg"],
  "avg_price": 5200.00,
  "low_price": 4800.00,
  "high_price": 5800.00
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | int | 是 | 用户 ID（服务端会从 JWT 获取真实值，但 JSON 中仍需传） |
| category | string | 是 | 品类 |
| brand | string | 是 | 品牌 |
| model | string | 是 | 型号 |
| condition | string | 是 | 成色描述 |
| image_urls | string[] | 否 | 图片 URL 数组 |
| avg_price | float\|null | 否 | 市场均价 |
| low_price | float\|null | 否 | 市场最低价 |
| high_price | float\|null | 否 | 市场最高价 |

### SSE 事件流

该接口返回 **`text/event-stream`**，前端需要用 `fetch` + `ReadableStream` 或 `EventSource` 读取。

**事件类型**：

| SSE type | 说明 | 数据格式 |
|----------|------|----------|
| `start` | 生成开始 | `{"type":"start","status":"started","message":"开始生成文案..."}` |
| `content` | 流式文本片段 | `{"type":"content","content":"这是一台"}` |
| `done` | 生成完成 | `{"type":"done","status":"success","message":"文案生成完成"}` |
| `error` | 生成失败 | `{"type":"error","status":"error","code":"GENERATION_ERROR","message":"错误描述"}` |

**完整 SSE 流示例**:
```
data: {"type":"start","status":"started","message":"开始生成文案..."}

data: {"type":"content","content":"📱"}

data: {"type":"content","content":" **Apple iPhone 15 Pro**"}

data: {"type":"content","content":"\n\n💰 售价：**5200元**\n\n"}

...

data: {"type":"done","status":"success","message":"文案生成完成"}
```

### 前端接收示例（Vue/JS）

```js
// 使用 fetch + ReadableStream
const response = await fetch('/api/v1/generate', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify(payload)
})

const reader = response.body.getReader()
const decoder = new TextDecoder()
let buffer = ''

while (true) {
  const { done, value } = await reader.read()
  if (done) break

  buffer += decoder.decode(value, { stream: true })
  const lines = buffer.split('\n')
  buffer = lines.pop() || ''

  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const event = JSON.parse(line.slice(6))
      // 处理 event.type: 'start' | 'content' | 'done' | 'error'
    }
  }
}
```

> ⚠️ **重要**:
> - 不能用 `axios` 的普通请求来接收 SSE，必须用 `fetch` 或 `EventSource`
> - 一定要支持 `AbortController` 以允许用户中途取消生成
> - `content` 事件会高频推送（每个 token 一次），前端应做节流渲染（如 RAF 合并）

## 5. 图片处理

全链路图片处理：预处理 + 瑕疵检测 + 画框标注 + DeepSeek 智能定价。

```
POST /api/v1/process/image
```

**认证**: 需要（JWT Bearer）  
**速率限制**: 30 次/分钟  
**Content-Type**: `multipart/form-data`

**请求参数**:

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| image | File | 是 | - | 商品图片文件 |
| item_id | int | 否 | null | 关联的商品 ID（有则保存瑕疵数据到数据库） |
| category | string | 否 | "手机" | 品类 |
| brand | string | 否 | "Apple" | 品牌 |
| model | string | 否 | "iPhone 14 Pro" | 型号 |

**成功响应** (200):
```json
{
  "success": true,
  "data": {
    "annotated_base64": "data:image/png;base64,iVBORw0KGgo...",
    "defects": [
      {
        "type": "scratch",
        "type_cn": "划痕",
        "bbox": [120, 200, 180, 260],
        "confidence": 0.87
      },
      {
        "type": "dent",
        "type_cn": "磕碰",
        "bbox": [300, 150, 360, 210],
        "confidence": 0.92
      }
    ],
    "defect_count": 2
  },
  "saved_files": {
    "annotated": "static/uploads/annotated/20250627_143052_a1b2c3d4.png"
  },
  "preprocess_info": {
    "success": true,
    "area_ratio": 0.85,
    "message": "去噪+增强完成"
  },
  "file_info": {
    "original_filename": "iphone.jpg",
    "file_size": 2048576
  },
  "market_price": 5200.00,
  "price_fallback": false,
  "message": "处理完成，检测到 2 个瑕疵",
  "deepseek_data": [
    {
      "type": "scratch",
      "type_cn": "划痕",
      "bbox": [120, 200, 180, 260],
      "area": 3600,
      "area_ratio": 0.018,
      "confidence": 0.87,
      "severity": "moderate",
      "severity_label": "中度"
    }
  ],
  "price_suggestion": {
    "success": true,
    "estimated_price": 4680,
    "price_range": [4400, 4900],
    "deduction_detail": "屏幕划痕扣减 300 元，边框磕碰扣减 220 元",
    "suggestion": "建议定价 4680 元，性价比高于市场均价 10%"
  }
}
```

### 响应字段详解

**data.defects** — 瑕疵列表（前端展示用，不含严重程度）:
```json
{
  "type": "scratch",       // 英文类型：scratch/dent/stain/crack/peeling
  "type_cn": "划痕",        // 中文名称：划痕/磕碰/污渍/裂痕/掉漆
  "bbox": [120, 200, 180, 260],  // [x1, y1, x2, y2] 边界框
  "confidence": 0.87       // 置信度 0-1
}
```

**deepseek_data** — DeepSeek 定价用的结构化瑕疵数据（含严重程度）:
```json
{
  "type": "scratch",
  "type_cn": "划痕",
  "bbox": [120, 200, 180, 260],
  "area": 3600,             // 像素面积
  "area_ratio": 0.018,      // 占图片面积比例
  "confidence": 0.87,
  "severity": "moderate",   // severe/moderate/minor/slight
  "severity_label": "中度"  // 重度/中度/轻度/轻微
}
```

**price_suggestion** — DeepSeek AI 定价建议（仅在检测到瑕疵时返回）:
```json
{
  "success": true,
  "estimated_price": 4680,          // 建议售价
  "price_range": [4400, 4900],      // 建议价格区间
  "deduction_detail": "...",        // 扣款明细说明
  "suggestion": "建议定价 4680 元..." // 综合建议文案
}
```

> ⚠️ **重要**:
> - `annotated_base64` 是 Base64 编码的 PNG 图片（可直接用于 `<img src>`），标注了瑕疵边界框
> - `price_fallback=true` 表示数据库中没查到行情价，使用了默认值 5000 元，应向用户提示
> - 如果传了 `item_id`，瑕疵数据会自动保存到数据库

## 注意事项与最佳实践

### 🔐 认证相关

1. **Token 拦截器**: 在 Axios 请求拦截器中自动附加 `Authorization` Header（当前 frontend 代码已实现）
2. **401 全局处理**: 在 Axios 响应拦截器中检测 401 → 清除 token → 跳转登录页（已实现）
3. **Token 过期**: 默认 30 分钟，生产环境建议增加 refresh token 机制或延长过期时间
4. **FormData 不走 JSON 拦截器**: 当前拦截器已判断 `config.data instanceof FormData` 时跳过 Content-Type 设置

### 📡 SSE 流式生成

1. **必须用 fetch ReadableStream**：不能使用 axios 的普通请求
2. **必须支持取消**: 实现 AbortController，允许用户中途停止生成
3. **必须处理异常**: 监听 `type: "error"` 事件，并显示错误信息
4. **渲染性能**: `content` 事件高频触发（每个 token 一次），用 `requestAnimationFrame` 合并 DOM 更新
5. **Nginx 配置**: 如有反向代理，需关闭缓冲（`proxy_buffering off;`）
6. **重试禁止**: SSE 失败后不要自动重试，由用户手动触发

### 📷 图片处理

1. **图片大小限制**: 最大 10MB，超出会被后端拒绝（400）
2. **支持格式**: JPG、PNG、WebP（后端用 PIL 校验）
3. **图片 URL 拼接**: 后端返回的图片路径可能是相对路径（如 `/static/uploads/xxx.jpg`），前端需拼上 Base URL。商城/历史接口已自动处理为完整 URL，但 extract 接口返回的是相对路径。
4. **Base64 图片**: `annotated_base64` 字段直接可用于 `<img :src="data.annotated_base64" />`

### 🏷️ 数据字段

1. **condition vs item_condition**: 统一使用 `condition` 字段（前端兼容属性 `item_condition` 仍存在但标记为废弃）
2. **price 类型**: 所有价格字段（`suggested_price`, `avg_price` 等）均为 `float`/`number` 类型，可能需要前端做 `.toFixed(2)` 展示
3. **category 枚举**: 目前系统支持：手机、笔记本、平板、耳机、手表、外设（可在 `market_prices` 表中扩展）

### ⚡ 性能与限制

1. **速率限制**: 务必处理 429 错误，根据 `Retry-After` 响应头做退避等待
2. **extract 接口较慢**: 涉及图片上传 + 预处理 + Qwen-VL API 调用，预计耗时 2-5 秒，前端应显示 loading 状态
3. **generate 接口流式返回**: 首个 token 可能需要 1-3 秒（LLM 首字延迟），后续持续推送
4. **process/image 接口耗时**: 涉及预处理 + YOLO 瑕疵检测 + 行情查询 + DeepSeek 定价，预计 3-8 秒

### 🖼️ 图片 URL 处理规则

```js
// 拼接规则（已在 history/market 接口中自动处理）
// 如果 image_url 以 http:// 或 https:// 开头 → 直接使用
// 如果 image_url 以 /static/ 开头 → 拼接 BASE_URL
// 否则 → 使用 get_static_url() 处理
```

### 🧩 发布流程建议

前端推荐的完整发布流程：

```
1. 用户上传图片 → POST /api/v1/extract → 获取 category/brand/model/condition + image_url
2. 用户确认/修正商品信息
3. 查行情价格 → GET /api/v1/price?brand=xxx&model=xxx → 获取 avg_price 等
4. [可选] 瑕疵检测 → POST /api/v1/process/image → 获取 defects + price_suggestion
5. 流式生成文案 → POST /api/v1/generate (SSE) → 流式展示生成结果
6. 用户确认文案 → POST /api/v1/history/save → 保存发布记录
```

### 🛡️ 安全注意事项（前端）

1. **所有认证接口的用户身份由 JWT 决定**，不要信任客户端传来的 `user_id`
2. **敏感操作（下架/删除/修改）只有所有者能执行**，后端已做校验
3. **密码不存储在 localStorage**，只存储 JWT token
4. **图片上传前做客户端预校验**：格式、大小（可减少无效请求）
5. **生产环境启用 HTTPS**，否则 JWT token 可能被中间人截获

### 📋 状态码速查

| 状态码 | 含义 | 前端处理 |
|--------|------|----------|
| 200 | 成功 | 正常处理 |
| 400 | 请求参数错误 | 显示 `detail` 中的错误信息 |
| 401 | 未认证/Token 过期 | 清除 token → 跳转登录 |
| 403 | 无权限 | 提示"无权操作此商品" |
| 404 | 资源不存在 | 提示用户 |
| 422 | 请求体校验失败 | FastAPI 自动返回字段级错误 |
| 429 | 频率限制 | 等待 `Retry-After` 秒后重试 |
