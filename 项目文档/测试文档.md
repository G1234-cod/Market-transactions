## 一、测试环境准备

### 1.1 依赖

测试相关依赖已包含在 `环境依赖/requirements.txt` 中：

| 包 | 版本 | 用途 |
|------|------|------|
| pytest | 8.3.4+ | 测试框架 |
| pytest-asyncio | 0.24.0+ | 异步测试支持 |
| pytest-cov | 6.0.0+ | 测试覆盖率 |

### 1.2 环境要求

运行测试前需要：
- MySQL 数据库已创建（`market_transactions`）
- 后端 `.env` 已配置（尤其是数据库连接信息）
- 虚拟环境已激活

### 1.3 配置文件

`backend/test/conftest.py` 是 pytest 的配置文件，提供共享的 fixtures。

## 二、运行测试

### 2.1 运行全部测试

```cmd
cd backend
pytest test/ -v
```

### 2.2 运行单个测试文件

```cmd
cd backend
pytest test/test_auth.py -v
```

### 2.3 运行特定测试函数

```cmd
cd backend
pytest test/test_auth.py::test_register -v
```

### 2.4 带覆盖率报告

```cmd
cd backend
pytest test/ -v --cov=app --cov-report=term-missing
```

### 2.5 跳过需要外部 API 的测试

部分测试需要真实的 API Key（DeepSeek、Qwen），在没有 Key 的环境下可以跳过：

```cmd
cd backend
pytest test/ -v -k "not extract and not generate and not deepseek and not qwen and not search"
```

只运行不依赖外部 API 的测试（认证、数据库、YOLO 检测）。

## 四、编写新测试

### 4.1 目录规范

```
backend/test/
├── __init__.py
├── conftest.py          # 共享 fixtures
├── test_auth.py         # 按模块划分
├── test_db.py
└── ...
```

### 4.2 命名规范

- 文件名：`test_<模块名>.py`
- 函数名：`test_<功能描述>`

### 4.3 使用 Fixtures

`conftest.py` 中提供了共享的 fixtures：

```python
# 异步 HTTP 客户端
@pytest.fixture
async def client():
    async with AsyncClient(app=app, base_url="http://test") as ac:
        yield ac

# 认证头
@pytest.fixture
async def auth_headers(client):
    # 先注册登录，获取 JWT token
    ...
    return {"Authorization": f"Bearer {token}"}
```

### 4.4 异步测试

使用 `pytest.mark.asyncio` 标记异步测试函数：

```python
import pytest

@pytest.mark.asyncio
async def test_my_feature(client, auth_headers):
    response = await client.post("/api/v1/my-endpoint", headers=auth_headers)
    assert response.status_code == 200
```

### 4.5 跳过条件

```python
import pytest

@pytest.mark.skip(reason="需要 DeepSeek API Key")
def test_needing_api_key():
    ...
```
