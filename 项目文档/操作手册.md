## 一、环境准备

| 软件 | 最低版本 | 检查命令 | 说明 |
|------|---------|---------|------|
| Python | 3.10 | `python --version` | 安装时勾选 "Add to PATH" |
| Node.js | 18+ | `node --version` | 前端构建需要 |
| MySQL | 8.0+ | `mysql --version` | 记住 root 密码 |
| Docker | 20+ | `docker --version` | 用于运行 Qdrant |

> 确保 MySQL 服务已启动。如未启动，以管理员身份运行：`net start MySQL80`

## 二、配置环境变量

```cmd
copy backend\.env.example backend\.env
```

编辑 `backend\.env`，**必须修改以下 4 项**：

```ini
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx     # https://platform.deepseek.com 注册获取
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxx    # https://dashscope.aliyun.com 注册获取
DB_PASSWORD=你的MySQL密码
SECRET_KEY=随机30个字符（JWT 签名密钥）
```

> 不填 API Key 不影响登录和商城浏览，但图片识别和文案生成会报错。

## 三、创建数据库

### 3.1 登录 MySQL 并执行建表脚本

```cmd
mysql -u root -p
```

进入 MySQL 命令行后：

```sql
SOURCE database/schema.sql;
SHOW TABLES;
```

应看到以下 11 张表：

```
ai_audit_logs         hard_cases           model_disagreements
category_brands       market_prices        model_metrics
feature_vectors       notifications        price_history
published_items       users
```

### 3.2 导入种子行情数据

```cmd
cd backend
python import_seed.py
```

## 四、安装 Python 依赖

```cmd
cd 环境依赖
pip install -r requirements.txt
python torchzidong.py
```

该脚本会自动检测 CUDA 版本并安装匹配的 PyTorch。下载慢可换清华源：

```cmd
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### 验证 GPU（可选）

```cmd
python -c "import torch; print(torch.cuda.is_available())"
```

`True` = GPU 可用，`False` = CPU 模式（较慢但可用）。

## 五、启动 Qdrant 向量数据库

```cmd
docker run -d -p 6333:6333 -v "%cd%\backend\data\qdrant:/qdrant/storage" qdrant/qdrant:latest
```

验证：浏览器打开 http://localhost:6333，看到 Qdrant 管理界面即成功。

## 六、启动后端服务

```cmd
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

看到以下输出说明成功：

```
INFO: Uvicorn running on http://0.0.0.0:8000
INFO: ✅ MySQL 连接池创建成功
INFO: ✅ 应用启动完成
```

验证：
- API 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

### 一键启动后端（含 Qdrant + 索引）

```cmd
cd backend
python run-backend.py                 # 完整启动
python run-backend.py --skip-qdrant   # 跳过 Qdrant
python run-backend.py --no-browser    # 不自动打开浏览器
```

## 七、安装并启动前端

```cmd
cd frontend
npm install
npm run dev
```

> npm 慢的话：`npm config set registry https://registry.npmmirror.com`

验证：浏览器打开 http://localhost:5173

## 八、一键启动全栈（推荐）

项目提供了 `run.py`，可一键启动后端 + 前端：

```cmd
python run.py                    # 完整启动（自动启动 Qdrant）
python run.py --skip-qdrant      # 跳过 Qdrant 启动
python run.py --no-browser       # 不自动打开浏览器
```

脚本会自动：启动后端 → 等待就绪 → 启动前端 → 打开浏览器。按 `Ctrl+C` 停止全部服务。

## 九、日常使用流程

### 9.1 发布商品

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① 上传 | 点击或拖拽上传商品图片 | Qwen-VL 自动识别品类/品牌/型号/成色 |
| ② 确认 | 查看并修正识别结果，可选"查价"和"瑕疵检测" | 查价返回市场行情，瑕疵检测返回标注图 |
| ③ 生成 | 点击"生成文案" | SSE 流式输出，打字机效果逐字出现 |
| ④ 发布 | 确认文案，点击"发布" | 商品进入商城，自动加入以图搜图索引 |

### 9.2 浏览商城

打开"商城"页面（无需登录）：
- 浏览所有已发布商品
- 关键词搜索（匹配标题和描述）
- 品类筛选（手机/笔记本/平板等）

### 9.3 管理我的发布

打开"历史"页面（需登录）：
- 查看草稿/已发布/已下架商品
- 发布草稿 / 下架商品

### 9.4 以图搜图

在商城页上传图片或输入文字描述，找到视觉上最相似的商品。

## 十一、常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| `uvicorn` 找不到 | 未激活虚拟环境 | `venv\Scripts\activate` |
| `Can't connect to MySQL` | MySQL 服务未启动 | `net start MySQL80` |
| `Access denied for user` | 密码错误 | 检查 `.env` 中 `DB_PASSWORD` |
| 识别/生成失败 | API Key 未配置或余额不足 | 检查 `.env` 中 `DEEPSEEK_API_KEY` 和 `DASHSCOPE_API_KEY` |
| 搜索报 500 | Qdrant 未启动 | `docker run -d -p 6333:6333 qdrant/qdrant` |
| 瑕疵检测返回 0 个 | 模型未训练 | 需要先训练 `defect_best.pt`，见 [未完成功能.md](未完成功能.md) |
| `npm install` 失败 | Node.js 版本过低 | 需要 Node.js 18+ |
| 前端白屏 | 后端未启动 | 检查 http://localhost:8000/health |
| pip 安装失败 | 网络问题 | 换清华源：`pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple` |
| YOLO 模型加载失败 | 权重文件缺失 | `yolov8n.pt` 会从 ultralytics 自动下载，或手动放置到 `backend/` |
| Docker 无法启动 | Docker Desktop 未运行 | 启动 Docker Desktop 后重试 |
