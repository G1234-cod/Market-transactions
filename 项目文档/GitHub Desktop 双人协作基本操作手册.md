## 一、准备工作

### 1.1 安装和登录

**两人都需要：**

1. 安装 GitHub Desktop
    
2. 登录自己的 GitHub 账号
    
3. 安装 Trae（或 VS Code）作为代码编辑器
    

### 1.2 克隆仓库到本地

1. 打开 GitHub Desktop
    
2. 点击 **`文件`** → **`克隆仓库`**
    
3. 选择 **`URL`** 标签
    
4. 输入仓库地址：`https://github.com/你的用户名/Market-transactions-main.git`
    
5. 选择本地存放路径（如 `D:\GitHub\Market-transactions-main`）
    
6. 点击 **`克隆`**
    

### 1.3 核心概念

|概念|说明|
|---|---|
|`main`|主分支，存放稳定代码，**不能直接修改**|
|你的分支|你的个人开发分支，**可以随便改**|
|队友分支|队友的个人开发分支，**可以随便改**|

## 二、创建分支

### 2.1 创建自己的开发分支

**每人各自操作一次：**

1. 在 GitHub Desktop 顶部，点击 **`当前分支`**
    
2. 点击 **`新建分支`**
    
3. 输入分支名：
    
    - 你：`feature/NXF-LGTM`
        
    - 队友：`feature/G1234-cod`
        
4. **来源** 选择 `main`
    
5. 点击 **`创建分支`**
    
6. 点击 **`发布分支`**（推送到 GitHub）
    

### 2.2 确认当前分支

在 GitHub Desktop 顶部，**`当前分支`** 旁边显示的就是你所在的分支：

|显示|含义|
|---|---|
|`main`|你在主分支|
|`feature/NXF-LGTM`|你在自己的分支|
|`feature/G1234-cod`|你在队友的分支|

## 三、日常开发流程

### 3.1 每天开始工作前

1. 确认你在自己的分支上（如 `feature/NXF-LGTM`）
    
2. 点击 **`拉取 origin`**
    
3. 点击 **`分支`** → **`合并到当前分支...`**
    
4. **来源** 选择 `main`
    
5. 点击 **`合并`**
    
6. 点击 **`推送 origin`**
    

### 3.2 切换分支

点击顶部 **`当前分支`** 下拉菜单 → 选择你要切换的分支。

## 四、提交与推送

### 4.1 提交更改

1. 在 Trae 中修改代码，保存（`Ctrl + S`）
    
2. 切换回 GitHub Desktop
    
3. 左侧 **`更改`** 列表显示所有修改过的文件
    
4. 勾选要提交的文件（或全选）
    
5. 在 **`摘要`** 框中写清楚修改内容
    
6. 点击 **`提交到 [你的分支名]`**
    

### 4.2 推送到 GitHub

提交完成后，点击顶部 **`推送 origin`**。

### 4.3 完整流程

text

写代码 → 保存 → 提交 → 推送（反复循环）

## 五、同步队友代码

### 5.1 把主分支同步到你分支

当队友合并了代码到 `main`，你需要同步到自己的分支：

1. 确保你在自己的分支上
    
2. 点击 **`拉取 origin`**
    
3. 点击 **`分支`** → **`合并到当前分支...`**
    
4. **来源** 选择 `main`
    
5. 点击 **`合并`**
    
6. 点击 **`推送 origin`**
    

### 5.2 把队友分支同步到你分支

当你需要队友的最新代码时：

1. 确保你在自己的分支上
    
2. 点击 **`拉取 origin`**
    
3. 点击 **`分支`** → **`合并到当前分支...`**
    
4. **来源** 选择队友的分支（如 `feature/G1234-cod`）
    
5. 点击 **`合并`**
    
6. 点击 **`推送 origin`**
    

### 5.3 查看队友代码（不合并）

1. 点击顶部 **`当前分支`** 下拉菜单
    
2. 选择队友的分支
    
3. 浏览文件
    
4. 查看完后，切回自己的分支
    

## 六、合并到主分支

### 6.1 什么时候可以合并？

**前提：** 你的功能**完全做好、测试通过**后，才能合并到 `main`。

### 6.2 合并步骤

1. 点击 **`当前分支`** → 切换到 `main`
    
2. 点击 **`拉取 origin`**
    
3. 点击 **`分支`** → **`合并到当前分支...`**
    
4. **来源** 选择你的开发分支（如 `feature/NXF-LGTM`）
    
5. 点击 **`合并`**
    
6. 点击 **`推送 origin`**
    

### 6.3 重要原则

|规则|说明|
|---|---|
|不要同时合并|一个人合并完推送后，另一个人再拉取再合并|
|合并前先拉取|确保 `main` 是最新版本|
|不要强制推送|永远不要用 `--force` 推送到 `main`|

## 七、解决冲突

### 7.1 什么时候会发生？

你们俩改了**同一个文件的同一行代码**。

### 7.2 冲突文件长什么样

text

<<<<<<< HEAD
你的代码
=======
队友的代码
>>>>>>> feature/G1234-cod

### 7.3 解决步骤

1. 在 Trae 中打开冲突文件
    
2. 决定保留哪个版本（或自己重写）
    
3. 删掉所有 `<<<<<<<`、`=======`、`>>>>>>>` 标记
    
4. 保存文件
    
5. 回到 GitHub Desktop，勾选冲突文件
    
6. 点击 **`提交合并`**
    
7. 点击 **`推送 origin`**
    

## 八、常用快捷键

|快捷键|作用|
|---|---|
|`Ctrl + Shift + A`|在外部编辑器中打开项目|
|`Ctrl + Shift + F`|在文件管理器中显示项目|
|`Ctrl + Shift + G`|在 GitHub 网页上打开仓库|
|`Ctrl + Alt + P`|预览 Pull Request|
|`Ctrl + P`|推送（有提交待推送时）|

## 九、常见问题

### 9.1 看不到队友的分支

text

点击 拉取 origin → 再点 当前分支 下拉菜单

### 9.2 推送失败

text

点击 拉取 origin → 再点击 推送 origin

### 9.3 不小心在 main 上改了代码

1. 点击 **`当前分支`** → **`新建分支`**
    
2. 输入分支名（如 `feature/紧急修复`）
    
3. 点击 **`发布分支`**
    
4. 按正常流程合并回 `main`
    

### 9.4 想放弃本地所有修改

点击 **`存储库`** → **`放弃所有更改...`**

### 9.5 合并时出现冲突

按 **第七节** 的步骤操作。

## 十、一句话总结

> **你和队友各自在独立分支上开发，每天从 main 同步最新，完成后推送到各自分支，功能完成后合并到 main。需要队友代码时直接合并队友分支。**
