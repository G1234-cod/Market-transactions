# 智能二手商品发布助手 —— 系统功能完整描述


## 📋 系统功能完整描述


### 一、用户系统

| 功能 | 说明 |
|:---|:---|
| 注册 | 用户创建账号，PBKDF2 加密存储密码 |
| 登录 | 用户名密码验证，返回 JWT Token（30 分钟有效期） |
| 认证 | JWT Bearer Token 统一认证 |
| 用户状态 | 支持 active/inactive/locked 状态 |


### 二、商品发布流程

```
上传图片 → 预处理 → 双模型识别 → 瑕疵检测 → 行情查询 → 文案生成 → 发布
```

| 步骤 | 功能 | 说明 |
|:---|:---|:---|
| 1 | 图片上传 | 单张图片，支持 JPG/PNG/WEBP，≤10MB |
| 2 | 图片预处理 | 尺寸统一（448×448）、去背景（rembg）、去噪（OpenCV） |
| 3 | 多物体判断 | 自动识别并选择占比最大的物体 |
| 4 | 物品识别 | 双模型并行（YOLOv8 + Qwen-VL-Max） |
| 5 | 识别结果 | 品类、品牌、型号、成色（结构化 JSON） |
| 6 | 双模型比对 | 结果不一致时自动保存到 `hard_cases` 错题本 |
| 7 | 瑕疵检测 | YOLOv8 检测：划痕/磕碰/污渍/裂痕/掉漆 |
| 8 | 画圈标注 | 不同形状+颜色代表不同程度（不告诉用户） |
| 9 | 行情查询 | 三层匹配策略（精确→模糊→空） |
| 10 | 文案生成 | DeepSeek SSE 流式生成（打字机效果） |
| 11 | 智能定价 | DeepSeek 结合瑕疵程度 + 市场行情 |
| 12 | 自动填表 | 标题、描述、价格自动填入 |
| 13 | 保存草稿 | 随时保存，防止数据丢失 |
| 14 | 发布商品 | 确认后发布到商城，自动加入 Qdrant 索引 |


### 三、瑕疵检测与标注

| 功能 | 说明 |
|:---|:---|
| 瑕疵类型 | 划痕（scratch）、磕碰（dent）、污渍（stain）、裂痕（crack）、掉漆（peeling） |
| 程度分级 | 重度（红色圆形）/ 中度（橙色矩形）/ 轻度（黄色多边形）/ 轻微（蓝色虚线框） |
| 程度隐藏 | 程度信息不展示给用户，仅供内部定价使用 |
| 标注图 | 在商品图上用不同形状和颜色标出瑕疵位置 |
| 结构化数据 | 瑕疵位置、类型、程度 → 发送给 DeepSeek 定价 |
| 瑕疵统计 | 自动统计瑕疵数量和严重程度分布 |


### 四、价格与定价

| 功能 | 说明 |
|:---|:---|
| 行情查询 | 三层匹配策略：精确匹配 → 模糊匹配（difflib > 0.7）→ 返回空 |
| 市场数据 | 品牌+型号 → 均价/最低价/最高价 |
| DeepSeek 定价 | 结合瑕疵程度和市场行情，输出建议售价 + 定价理由 |
| 价格调整 | 重度损伤降价 15-30%，中度 5-15%，轻度 0-5% |
| 定价理由 | 可解释性输出，增强用户信任 |


### 五、以图搜图

| 功能 | 说明 |
|:---|:---|
| 以图搜图 | 上传图片 → CLIP 提取 512 维特征向量 → Qdrant 检索 → 返回相似商品 |
| 以文搜图 | 输入文字描述 → CLIP 文本编码 → Qdrant 检索 → 返回匹配商品 |
| 自动索引 | 商品发布时自动提取特征向量入库 |
| 手动索引 | 支持批量索引脚本 `index_items.py --all` |
| 索引维护 | 下架时自动从 Qdrant 删除 |
| 相似度排序 | COSINE 相似度，返回 Top-K 结果 |


### 六、商城

| 功能 | 说明 |
|:---|:---|
| 商品列表 | 展示所有已发布商品（`status='published'`） |
| 关键词搜索 | 模糊匹配商品标题和描述 |
| 品类筛选 | 按品类精确筛选 |
| 点赞 | 用户对商品点赞/取消点赞（显式事务保证一致性） |
| 商品详情 | 查看商品信息 + 瑕疵标注图 |
| 访问统计 | 自动记录浏览次数（views） |
| 无需登录 | 商城浏览无需认证 |


### 七、发布历史

| 功能 | 说明 |
|:---|:---|
| 历史列表 | 查看当前用户所有发布记录 |
| 状态筛选 | 按草稿/已发布/已下架筛选 |
| 下架 | 下架已发布的商品（仅商品所有者） |
| 重新上架 | 将下架商品重新发布（仅商品所有者） |
| 草稿箱 | 保存未完成的草稿，可继续编辑发布 |
| 权限校验 | 操作他人商品返回 403 |


### 八、AI 审计日志

| 功能 | 说明 |
|:---|:---|
| 日志记录 | 记录每次 AI 调用（Qwen/YOLO/DeepSeek/价格查询） |
| 执行耗时 | 记录每次调用的毫秒数 |
| 状态追踪 | SUCCESS / FAILED |
| 错误记录 | 失败时记录完整错误信息 |
| JSON 存储 | 原始响应以 JSON 格式存储（避免双编码） |
| 可追溯 | 全链路可追溯，问题排查 |


### 九、数据飞轮（持续进化）

| 功能 | 说明 |
|:---|:---|
| 错误数据收集 | 双模型结果不一致时自动保存图片 + 标签文件 |
| 错题本存储 | `hard_cases` 表，唯一索引防止重复 |
| 每周迭代训练 | `weekly_train.py` 自动拉取错误数据微调模型 |
| 模型自动更新 | 训练完成后自动替换 `best.pt` |
| 爬虫采集 | `crawler.py` 从二手平台采集公开数据补充训练集 |
| 数据备份 | 数据库写入失败时保存到本地文件 |


### 十、健康检查与监控

| 功能 | 说明 |
|:---|:---|
| 全局健康 | `/health` — 数据库连接状态 |
| YOLO 健康 | `/api/v1/yolo/health` — 模型加载状态 |
| 搜索健康 | `/api/v1/search/health` — CLIP + Qdrant 状态 |
| 处理健康 | `/api/v1/process/health` — 瑕疵检测 + DeepSeek 状态 |
| 索引统计 | `/api/v1/search/stats` — Qdrant 向量数量 |
