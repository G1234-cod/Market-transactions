## 📋 系统功能完整描述


### 一、用户系统

| 功能 | 说明 |
|:---|:---|
| 注册 | 用户创建账号，密码加密存储 |
| 登录 | 用户名密码验证，返回用户信息 |
| 用户状态 | 支持 active/inactive/locked 状态 |


### 二、商品发布流程

```
上传图片 → 预处理 → 识别 → 瑕疵检测 → 自动填表 → 发布
```

| 步骤 | 功能 | 说明 |
|:---|:---|:---|
| 1 | 图片上传 | 支持单张图片上传 |
| 2 | 图片预处理 | 尺寸统一（448×448）、去背景（rembg）、去噪 |
| 3 | 多物体判断 | 自动识别并选择占比最大的物体 |
| 4 | 物品识别 | 双模型并行（YOLO + Qwen），识别品类/品牌/型号 |
| 5 | 双模型比对 | 结果不一致时保存错误数据到 `hard_cases` |
| 6 | 瑕疵检测 | 检测划痕/磕碰/污渍/裂痕/掉漆 |
| 7 | 画圈标注 | 不同形状+颜色代表不同程度（不告诉用户） |
| 8 | 自动填表 | 标题、描述、价格自动填入表单 |
| 9 | 保存草稿 | 随时保存，防止数据丢失 |
| 10 | 发布商品 | 确认后发布到商城 |


### 三、瑕疵检测与标注

| 功能 | 说明 |
|:---|:---|
| 瑕疵类型 | 划痕、磕碰、污渍、裂痕、掉漆、变形 |
| 程度分级 | 重度（红色圆形）/ 中度（橙色矩形）/ 轻度（黄色多边形）/ 轻微（蓝色虚线） |
| 程度隐藏 | 程度信息不展示给用户，仅供内部定价 |
| 标注图 | 在商品图上用不同形状和颜色标出瑕疵位置 |
| 结构化数据 | 瑕疵位置、类型、程度 → 发送给 DeepSeek 定价 |


### 四、价格与定价

| 功能 | 说明 |
|:---|:---|
| 行情查询 | 根据品牌+型号查询市场均价/最低价/最高价 |
| DeepSeek 定价 | 结合瑕疵程度和市场行情，给出具体建议售价 |
| 价格调整 | 重度损伤降价 15-30%，中度 5-15%，轻度 0-5% |


### 五、以图搜图

| 功能 | 说明 |
|:---|:---|
| 以图搜图 | 上传图片 → CLIP 提取特征 → FAISS/Qdrant 检索 → 返回相似商品 |
| 以文搜图 | 输入文字描述 → 搜索匹配商品 |
| 自动索引 | 商品上架时自动提取特征向量入库 |


### 六、商城

| 功能 | 说明 |
|:---|:---|
| 商品列表 | 展示所有已发布商品 |
| 搜索 | 关键词搜索商品标题/描述 |
| 分类筛选 | 按品类筛选商品 |
| 点赞 | 用户对商品点赞/取消点赞 |
| 商品详情 | 查看商品信息 + 瑕疵标注图 |


### 七、发布历史

| 功能 | 说明 |
|:---|:---|
| 历史列表 | 查看所有发布记录 |
| 状态筛选 | 按草稿/已发布/已下架/已售出筛选 |
| 下架 | 下架已发布的商品 |
| 重新上架 | 将下架商品重新发布 |
| 草稿箱 | 保存未完成的草稿 |


### 八、AI 审计日志

| 功能 | 说明 |
|:---|:---|
| 日志记录 | 记录每次 AI 调用（Qwen/YOLO/DeepSeek） |
| 执行耗时 | 记录每次调用的毫秒数 |
| 状态追踪 | SUCCESS / FAILED |
| 错误记录 | 失败时记录错误信息 |


### 九、数据飞轮（持续进化）

| 功能 | 说明 |
|:---|:---|
| 错误数据收集 | 双模型结果不一致时自动保存图片+标签 |
| 每周迭代训练 | 每周用错误数据微调模型 |
| 爬虫采集 | 从二手平台采集公开数据补充训练集 |
